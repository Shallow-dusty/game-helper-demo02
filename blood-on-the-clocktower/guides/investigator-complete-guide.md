# 调查员（Investigator）深度完整手册

## 📚 角色基础信息

### Role Identity / 角色身份
- **英文名称**: Investigator
- **中文名称**: 调查员
- **阵营 (Team)**: 善良 (Good)
- **类型 (Type)**: 镇民 (Townsfolk)
- **版本 (Edition)**: Trouble Brewing (暗流涌动)

### Ability Description / 能力描述

**英文原文**:
> "You start knowing that 1 of 2 players is a particular Minion. (Or that zero are in play.)"

**中文翻译**:
> "游戏开始时，你会得知 2 名玩家中有 1 人是特定的爪牙。（或者得知场上没有爪牙。）"

## ⚙️ 能力机制详解

### Ability Mechanics / 运作机制

#### First Night Only / 仅限第一夜
调查员的能力**仅在第一夜触发一次**，之后不再获得新信息。这使得调查员成为一个典型的"开局信息角色"，需要在游戏早期最大化利用这一次性信息。

#### Information Received / 获得的信息

说书人会在第一夜唤醒调查员，并提供以下信息之一：

1. **标准信息**: 指向两名玩家，并展示一个爪牙角色标记
   - 含义：这两人中**有且仅有一人**是该爪牙
   - 你**不知道具体是哪一个**
   - 可能的爪牙类型：毒师 (Poisoner)、间谍 (Spy)、猩红女郎 (Scarlet Woman)、男爵 (Baron)

2. **零爪牙信息**: 展示数字 "0"
   - 含义：本局游戏**没有任何爪牙**
   - 这是极其强大的信息，因为标准配置应该有爪牙
   - 通常意味着出现了特殊情况（如醉酒者占据了爪牙位置）

#### Special Rules / 特殊规则

##### 关于醉酒者 (The Drunk)
这是调查员最重要的机制细节：

- 如果调查员自己是**醉酒者 (Drunk)**
- 调查员会收到**完全错误的信息**
- 醉酒者不知道自己的真实身份，以为自己真的是调查员

**示例**:
```
场景：你是醉酒者，以为自己是调查员
说书人可能告诉你：
- 指向 Alice 和 Bob
- 展示 "Poisoner" 标记
✗ 现实：Alice 和 Bob 可能都不是毒师
✗ 甚至：场上可能根本没有毒师
```

##### 关于中毒状态
如果调查员第一夜被**下毒**（由毒师能力触发）：
- 可能收到**完全错误的信息**
- 可能被告知不存在的爪牙
- 可能被告知错误的玩家组合
- 说书人可以自由决定给予何种错误信息

**关键区别**:
```
醉酒者调查员：从游戏开始就是醉酒者，自己不知道
被下毒调查员：原本是真调查员，第一夜恰好被下毒
```

##### 关于"0"信息的特殊情况

**何时会收到"0"？**

1. **正常情况**: 场上确实没有爪牙
   - 这在标准 Trouble Brewing 中**极其罕见**
   - 需要特殊的角色组合（如多个醉酒者）

2. **异常情况**: 你被干扰（醉酒或中毒）
   - 说书人给你错误的"0"信息
   - 实际场上有爪牙

##### 信息的准确性保证

**如果你是清醒且未中毒的调查员**：
- 你的信息**100% 准确**
- 两名候选人中**确实有一人**是你看到的爪牙
- 不存在"部分正确"的情况

## 🎯 玩法策略

### As the Investigator / 作为调查员

#### 核心目标
1. **识别真实的爪牙** - 从两名候选人中找出邪恶玩家
2. **保护关键信息** - 决定何时公开、向谁公开、公开多少
3. **领导善良阵营** - 利用强力信息建立信任和指挥权
4. **验证自身状态** - 确认自己没有被干扰

### Opening Strategy / 开局策略

#### Scenario 1: 收到爪牙信息

**Day 1 Actions / 第一天行动**:

1. **深度观察候选人行为**
   ```
   观察要点：
   - 谁更主动发言？（爪牙可能试图建立信任）
   - 谁更沉默？（爪牙可能低调行事）
   - 谁的投票模式异常？
   - 谁在私下会议中表现可疑？
   - 谁试图转移话题或引导讨论？
   ```

2. **不要立即全盘公开**
   ```
   ❌ 错误做法：
   "我是调查员！Alice 和 Bob 中有一个是毒师！"

   ✓ 更好做法：
   - Day 1: 保持沉默，纯粹观察
   - Day 1-2: 私下接触可信玩家，建立小联盟
   - Day 2-3: 部分公开信息
   - Day 3+: 根据局势决定完全公开时机
   ```

3. **建立信息网络**
   - 私下找 2-3 个看起来可信的玩家
   - 交换信息（但不一定立即透露全部）
   - 寻找其他信息角色（占卜师、洗衣妇、图书管理员）
   - 交叉验证彼此的信息

4. **分析爪牙类型的含义**
   ```
   毒师 (Poisoner):
   - 说明有人的能力被干扰
   - 询问谁的信息看起来不合理
   - 找到被下毒的目标可以反推毒师

   间谍 (Spy):
   - 极其危险！间谍知道所有善良信息
   - 可能有人的角色宣称被间谍知晓
   - 警惕信息泄露过快的情况

   猩红女郎 (Scarlet Woman):
   - 恶魔死后的备用方案
   - 可能坐在恶魔旁边
   - 关注座位关系

   男爵 (Baron):
   - 增加外来者数量
   - 统计外来者宣称数量
   - 如果外来者过多，验证男爵存在
   ```

#### Scenario 2: 收到"0"信息

这是**极其强大且罕见**的信息！

**含义深度分析**:
```
✓ 场上没有任何爪牙角色
✓ 恶魔是孤军奋战
✓ 任何宣称是爪牙的玩家必然在撒谎
✓ 可能存在特殊配置（多醉酒者）
✓ 或者... 你自己被干扰了
```

**行动建议**:
```
Day 1-2:
- 保持完全沉默
- 观察是否有人宣称外来者（男爵效果）
- 观察是否有人行为像爪牙

Day 2-3:
- 如果有人宣称爪牙 → 记录下来
- 继续观察，不要打草惊蛇

Day 3-4:
- 选择时机公开"0"信息
- 揭露所有爪牙宣称者
- 建立领导地位

关键警告：
! 在公开前，确保自己不是醉酒者
! 寻找证据证明信息真实性
! 如果"0"信息与局势严重矛盾 → 考虑被干扰
```

### When to Reveal / 何时公开身份

#### 公开时机的战术选择

| 时机 | 优势 | 劣势 | 推荐场景 |
|------|------|------|----------|
| **第一天立即公开** | 快速建立信任和领导地位 | 成为击杀目标，失去观察优势 | 收到"0"且需要立即控场 |
| **第二天有选择公开** | 有观察数据支撑判断 | 可能被质疑为何延迟 | 已基本确定爪牙身份 |
| **第三天谨慎公开** | 充分观察，信息价值最大化 | 可能错过最佳时机 | 局势复杂，需更多数据 |
| **保持完全隐藏** | 避免成为目标，保持灵活性 | 无法充分利用信息优势 | 怀疑自己被干扰 |
| **临终公开** | 死前传递关键信息 | 浪费整个游戏期间的价值 | 邪恶势力占优，需保命 |

#### 最佳实践流程
```
推荐时间线：

Day 1 早晨:
- 倾听所有人的角色宣称
- 观察谁主动，谁被动
- 不透露任何信息

Day 1 下午:
- 私下接触 1-2 个可信玩家
- 建立小型信任圈
- 交换初步信息

Day 2 早晨:
- 根据候选人表现，初步判断
- 继续收集其他信息角色的情报

Day 2 下午:
- 决定是否部分公开
- 可以透露"我有爪牙信息"但不说具体

Day 3:
- 根据局势决定完全公开时机
- 如果爪牙已暴露特征 → 公开
- 如果仍不确定 → 继续隐藏

Day 4+:
- 如果还未公开，评估风险
- 可能需要在被杀前公开
```

### Advanced Techniques / 高级技巧

#### 技巧1: 压力测试法
```
目标：通过施加心理压力，观察候选人反应

步骤：
1. Day 1 私下对候选人 A 说：
   "我有一些关于你的信息，我们需要谈谈"

2. 观察反应：
   - 善良玩家：通常坦诚，愿意分享信息
   - 爪牙玩家：可能紧张，回避，或过度解释

3. Day 2 对候选人 B 重复测试

4. 对比两人反应差异 → 识别可疑者
```

#### 技巧2: 信息交叉验证网络
```
建立多角度验证体系：

与占卜师合作：
- 请求验证你的候选人之一
- 如果验证出恶魔 → 可能是恶魔伪装爪牙
- 如果验证出不是恶魔 → 缩小范围

与共情者合作：
- 查看候选人的邻居情况
- 如果共情者显示候选人旁边有邪恶 → 线索

与洗衣妇/图书管理员合作：
- 交叉对比信息
- 如果信息一致 → 互相验证真实性

与送葬者合作：
- 如果候选人之一死亡
- 送葬者可以确认其真实身份
- 验证你的信息准确性
```

#### 技巧3: 分阶段信息披露
```
原理：不一次性公开所有信息，而是分层释放

阶段 1 (Day 1):
"我是一个信息角色"
→ 观察谁紧张，谁好奇

阶段 2 (Day 2):
"我有关于邪恶阵营的信息"
→ 观察邪恶阵营的应对准备

阶段 3 (Day 2-3):
"我知道某两人中有一个是爪牙"
→ 观察候选人反应

阶段 4 (Day 3):
"是 Alice 和 Bob"
→ 观察两人的即时反应

阶段 5 (Day 3-4):
"是毒师"
→ 完整信息公开

优势：
✓ 每个阶段都收集新的行为数据
✓ 邪恶阵营难以提前布局
✓ 最大化信息价值
```

#### 技巧4: 假信息诱饵（高风险）
```
场景：你怀疑 Carol 是恶魔，但 Carol 不在你的候选人名单中

策略：
1. 公开虚假信息：
   "我是调查员，Carol 和 Dave 中一人是间谍"
   （实际你的信息是 Alice/Bob 中一人是毒师）

2. 观察 Carol 的反应：
   - 如果她疯狂否认 → 可能确实有问题
   - 如果她认领间谍 → 可能是善良被误导
   - 如果她异常冷静 → 可能是恶魔（知道你在撒谎）

3. 观察真实候选人（Alice/Bob）的反应：
   - 真爪牙可能松了一口气（压力转移）
   - 暴露微表情

4. Day 4-5 澄清真实信息：
   "抱歉，我之前测试大家，真实信息是..."

⚠️ 警告：
- 这会严重损害信任
- 仅在必要时使用
- 需要后期重建信任
```

#### 技巧5: 利用座位信息
```
爪牙类型 + 座位关系 = 额外线索

如果是猩红女郎：
- 她可能坐在恶魔附近（非必须但常见）
- 观察候选人的邻居是否可疑

如果是间谍：
- 间谍需要收集信息
- 可能坐在信息角色附近

如果是毒师：
- 观察谁的信息看起来被干扰
- 被下毒玩家的邻居可能是毒师

如果是男爵：
- 统计外来者数量
- 如果外来者数量符合男爵效果 → 验证
```

## ⚠️ 常见误区

### Mistakes to Avoid / 新手易犯错误

#### 误区1: 第一天全盘公开所有信息
```
错误示范：
"大家好！我是调查员！Alice 和 Bob 中有一个是毒师！大家一起来找！"

为什么这是错误的：
❌ 立即成为恶魔的首要击杀目标
❌ 失去观察候选人自然反应的机会
❌ 邪恶阵营有充足时间布局应对
❌ 间谍可以立即伪装
❌ 无法利用信息优势建立私密联盟
❌ 真爪牙可以编造完美的解释

正确做法：
✓ Day 1: 完全沉默，纯粹观察
✓ Day 1-2: 私下建立小联盟
✓ Day 2-3: 根据观察结果，选择性公开
✓ 保留完整信息直到最佳时机
```

#### 误区2: 过度信任自己的信息
```
错误思维：
"我是调查员，我的信息肯定是对的！Alice 一定是毒师！"

现实风险：
✓ 你可能是醉酒者（不自知）
✓ 你可能第一夜被下毒（收到错误信息）
✓ 说书人可能给了平衡性的误导信息（在合法范围内）
✓ 你可能误解了说书人的手势

正确心态：
✓ 始终保持"我可能被干扰"的警觉
✓ 寻找多重验证渠道
✓ 如果信息与局势严重不符 → 重新评估
✓ 愿意承认错误并调整策略
```

#### 误区3: 忽视"0"信息的强大威力
```
错误反应：
收到"0" → "哦，没有爪牙啊，那我没什么作用了" → 沉默整场

正确认识：
"0"信息 = 极强的战术武器！

价值：
✓ 可以揭穿所有爪牙伪装者
✓ 确认没有毒师（信息更可信）
✓ 确认没有间谍（安全分享信息）
✓ 确认没有猩红女郎（处决恶魔即胜利）
✓ 确认没有男爵（外来者数量标准）
✓ 建立强大的领导地位

行动：
✓ 保持沉默，等待有人宣称爪牙
✓ 在关键时刻公开"0"，揭穿伪装
✓ 领导善良阵营锁定恶魔
```

#### 误区4: 单纯依赖信息，不观察行为
```
错误做法：
只关注"Alice 和 Bob 中一人是毒师"
不观察他们的投票、发言、私聊行为

问题：
❌ 信息只是起点，不是终点
❌ 行为分析才能确定具体是谁
❌ 缺乏行为数据，信息价值减半

正确做法：
✓ 详细记录候选人的每一次投票
✓ 观察他们在私下会议中的表现
✓ 注意他们对不同话题的反应
✓ 观察他们与其他玩家的互动
✓ 综合信息 + 行为 = 准确判断
```

#### 误区5: 对候选人产生个人偏见
```
错误思维：
"我喜欢 Alice，她人很好，肯定不是爪牙"
"我不喜欢 Bob，他肯定是爪牙"

危险：
❌ 个人喜好影响理性判断
❌ 真爪牙可能是你喜欢的人
❌ 善良玩家可能行为可疑

正确心态：
✓ 保持客观和中立
✓ 基于证据，不是感觉
✓ 记住：这是游戏，不是现实
✓ 最好的朋友也可能是爪牙（在游戏中）
```

#### 误区6: 公开后不保护自己
```
错误行为：
Day 2 公开调查员身份 → Day 3 继续高调发言 → Night 3 被杀

问题：
❌ 公开后成为首要目标
❌ 没有寻求保护（僧侣、士兵）
❌ 没有传递信息给可信玩家
❌ 死后信息链断裂

正确做法：
✓ 公开前，建立信息传递链
✓ 公开后，立即联系僧侣/士兵
✓ 将完整信息告知 2-3 个可信玩家
✓ 即使你被杀，信息仍能传播
✓ 考虑"假死"策略（让恶魔以为你被保护）
```

#### 误区7: 死守错误判断
```
场景：
你认定 Alice 是爪牙
Day 3 新信息显示可能是 Bob
你仍坚持："不可能！肯定是 Alice！"

问题：
❌ 认知固化（Confirmation Bias）
❌ 忽视新证据
❌ 误导善良阵营
❌ 失去信任

正确做法：
✓ 保持开放思维
✓ "根据新信息，我需要重新评估..."
✓ 愿意公开承认"我之前可能判断错了"
✓ 根据最新数据调整策略
✓ 诚实 > 面子
```

### How to Avoid / 避免方法

**建立系统化的验证机制**
```
验证清单：
□ 找到至少 2 个其他信息角色
□ 交叉对比信息一致性
□ 观察候选人至少 2 天的行为
□ 寻找第三方证据（共情者、送葬者）
□ 如果信息冲突 → 考虑被干扰可能性
```

**保持自我质疑**
```
每天问自己：
1. 我的信息与其他人一致吗？
2. 候选人的行为符合我的判断吗？
3. 我有没有因为偏见影响判断？
4. 如果我被下毒，会是什么情况？
5. 我是否过度自信？
```

**灵活的公开策略**
```
不要预设"必须第 X 天公开"
根据以下因素动态决定：

✓ 局势紧迫性
✓ 候选人暴露程度
✓ 其他信息角色的发现
✓ 自己的安全状况
✓ 善良阵营的凝聚力

有时隐藏更有价值
有时立即公开更有效
没有固定公式
```

## 🎓 高级玩法

### Expert Strategies / 资深玩家技巧

#### 策略1: "双候选人压力循环"
```
原理：在两名候选人之间创造心理压力循环

Day 1:
私下对 Alice 说："我有一些关于你的信息"
→ 观察她的紧张程度

Day 2:
私下对 Bob 说："有人告诉我关于你的消息"
→ 观察他的反应

Day 2 晚上:
再次接触 Alice："我刚和 Bob 谈过，他说了一些有趣的事"
→ Alice 会猜测你知道什么

Day 3:
对比两人的累积反应：
- 谁更紧张？
- 谁试图反向打探消息？
- 谁表现更坦荡？

结论：
真爪牙在持续压力下更容易露出破绽
```

#### 策略2: "信息市场化"
```
原理：把信息当作交易筹码，换取其他玩家的情报

操作：
1. 不直接公开信息
2. 对可信玩家说：
   "我有爪牙信息，但我需要先知道你的情报"

3. 交换原则：
   - 对方先分享部分信息
   - 你分享部分信息
   - 逐步建立互信
   - 最终完全交换

优势：
✓ 确保对方也付出（增加真实性）
✓ 建立深层信任关系
✓ 获得更多信息
✓ 如果对方是邪恶 → 他们难以编造完整谎言

风险：
⚠️ 如果对方是间谍 → 你暴露信息
⚠️ 需要准确判断对方可信度
```

#### 策略3: "时间差情报战"
```
原理：向不同人公开不同版本的信息，观察谁泄露给谁

Day 2:
对 Carol 说："Alice 和 Bob 中一人是毒师"
对 Dave 说："Alice 和 Eve 中一人是间谍"
（实际信息：Alice 和 Bob 中一人是毒师）

Day 3:
观察：
- 如果有人知道"Alice 和 Eve"的版本 → 来自 Dave
- 如果邪恶阵营应对"间谍"话题 → Dave 可能是邪恶
- 追踪信息流向 → 识别邪恶网络

Day 4:
公开真实信息："抱歉，我之前在测试信息渠道"

⚠️ 极高风险策略：
- 严重破坏信任
- 仅在绝望情况使用
- 需要后期大量重建工作
```

#### 策略4: "反向中毒推理"
```
适用场景：你怀疑自己被下毒

操作：
1. 假设你的信息是错误的
2. 反向推理："如果我被下毒，谁是毒师？"

线索：
- 毒师第一夜就知道你是调查员
- 毒师可能坐在你附近（方便选择目标）
- 毒师希望你误导善良阵营

Day 3-4:
公开："我怀疑自己被下毒，我的信息可能是错的"
然后："但这意味着毒师第一夜就知道我的角色"
"谁有可能在第一夜知道？"

分析：
- 座位关系
- 谁的行为符合"下毒者"
- 谁在推动你的错误信息

结论：
将劣势（被下毒）转化为线索（识别毒师）
```

#### 策略5: "多重公开假设"
```
高级公开技巧：不是简单地说"是 A 和 B"，而是提供分析

公开方式：
"我是调查员，我看到 Alice 和 Bob 中一人是毒师。

基于我的观察：
- Alice 在 Day 1 的投票符合善良行为
- Bob 在私下会议中试图转移话题
- Alice 主动分享了她的信息
- Bob 对某些问题回避

我的初步判断：70% 认为是 Bob，30% 认为是 Alice

但我保持开放态度，欢迎其他信息角色帮忙验证"

优势：
✓ 展示你的理性分析过程
✓ 建立领导地位（不是独裁）
✓ 邀请合作，而非强制
✓ 保留调整空间
✓ 即使判断错误，也不会完全失信
```

#### 策略6: "座位几何学"
```
利用座位关系推理爪牙身份

如果是猩红女郎：
- 观察谁可能坐在恶魔附近
- 猩红女郎需要在恶魔死后接管
- 通常（不总是）坐在恶魔旁边或附近

如果是毒师：
- 毒师需要每晚选择目标
- 可能坐在信息角色密集的区域

如果是间谍：
- 间谍需要收集信息
- 观察座位关系对信息获取的便利性

如果是男爵：
- 男爵影响初始设置，座位相对随机
- 但观察谁与外来者有互动

技巧：
在 Day 1 绘制座位图
标注每个人的宣称角色
寻找几何模式
结合爪牙类型推理合理位置
```

## 🎭 说书人指引

### For Storytellers / 给说书人的建议

#### 选择哪两名玩家？

**原则1: 游戏平衡性**
```
✓ 优秀选择：
- 一名活跃玩家 + 一名普通玩家
  → 增加识别难度，但不至于无解

- 两名表达能力接近的玩家
  → 公平竞争

- 新手真爪牙 + 老手善良玩家
  → 给新手邪恶玩家一些挑战

✗ 应避免：
- 两名完全新手（可能快速暴露）
- 两名完全沉默的玩家（信息浪费）
- 明显能力不对等（不公平）
```

**原则2: 戏剧性和趣味性**
```
优秀选择示例：

如果真爪牙是毒师：
- 选择毒师 + 一名强力信息角色
  → 创造"我被下毒了吗？"的疑问
  → 增加心理博弈

如果真爪牙是间谍：
- 选择间谍 + 一名擅长伪装的善良玩家
  → 真假间谍对决

如果真爪牙是猩红女郎：
- 选择猩红女郎 + 坐在她附近的玩家
  → 利用座位信息增加复杂性

如果真爪牙是男爵：
- 选择男爵 + 一名外来者
  → 增加混淆（男爵制造外来者）
```

**原则3: 玩家体验**
```
考虑因素：

对于调查员玩家：
- 给予足够挑战，但不是无解
- 确保有观察和推理的空间
- 避免"过于明显"或"完全无法判断"

对于真爪牙玩家：
- 给予公平的伪装机会
- 不要通过选择让他立即暴露

对于另一名候选人（善良）：
- 给予展示清白的机会
- 不要让无辜玩家过于被动
```

#### 何时给予"0"信息？

**标准情况**:
```
场上确实没有爪牙时：
- 罕见配置（如多个醉酒者占据爪牙位）
- 特殊剧本设置

标准 Trouble Brewing 配置：
7 人局: 5 镇民, 0 外来者, 1 爪牙, 1 恶魔
通常不会出现"0 爪牙"
```

**特殊情况（调查员被干扰）**:
```
如果调查员是醉酒者：
✓ 可以给"0"（即使有爪牙）
✓ 可以给任何错误信息
✓ 目的：制造混乱

如果调查员第一夜被下毒：
✓ 可以给"0"（即使有爪牙）
✓ 可以指向两名都不是爪牙的玩家
✓ 可以指向正确爪牙 + 错误爪牙类型
```

**平衡考虑**:
```
"0"信息非常强大：
- 可以揭穿所有爪牙伪装
- 极大帮助善良阵营

建议：
- 仅在确实无爪牙时给予
- 或在调查员被干扰时给予（平衡）
- 不要随意给予真实的"0"信息
```

#### 如何展示信息？

**标准流程**:
```
第一夜，在"调查员信息"阶段：

1. 唤醒调查员（眼神接触确认）

2. 指向两名玩家：
   - 明确指向，确保调查员看清
   - 两次确认（指两遍）
   - 观察调查员点头确认

3. 展示爪牙角色标记：
   - 清晰展示
   - 保持3-5秒
   - 确保调查员识别

4. 让调查员睡下：
   - 确认调查员闭眼
   - 继续其他夜晚流程

⚠️ 常见错误：
✗ 指向不清（调查员没看清是谁）
✗ 角色标记展示太快
✗ 没有二次确认
```

**展示"0"的方式**:
```
方法1: 手势"0"
- 用手指比出"0"
- 明确展示

方法2: 摇头
- 摇头 + 手势表示"无"

方法3: 角色标记
- 准备一个"0"的数字标记
- 专门用于此类信息

⚠️ 确保调查员理解"0"的含义：
✓ 在游戏前说明"0"的意思
✓ 如果调查员看起来困惑，重复展示
```

#### 处理调查员被干扰的情况

**情况1: 调查员是醉酒者**
```
说书人的自由度：
✓ 完全自由选择给予何种错误信息
✓ 可以给"0"（即使有爪牙）
✓ 可以指向任意两名玩家
✓ 可以展示任意爪牙类型

建议：
- 给予"看起来合理"的错误信息
  （不要过于荒谬，保持游戏趣味）
- 考虑给予能制造最多混乱的信息
- 平衡游戏，不要让邪恶过于轻松获胜
```

**情况2: 调查员第一夜被下毒**
```
说书人操作：
1. 在"毒师"阶段，毒师选择调查员
2. 在"调查员信息"阶段：
   - 给予错误信息（同醉酒者处理）
   - 完全自由选择

3. 第二夜，如果调查员不再被下毒：
   - 调查员已经收到信息（第一夜）
   - 不再给予新信息
   - 调查员的错误信息永久存在
```

**平衡性建议**:
```
如果邪恶阵营过强：
- 给调查员准确信息
- 选择容易识别的候选人组合

如果善良阵营过强：
- 考虑第一夜下毒调查员
- 或选择难以区分的候选人组合

如果游戏平衡：
- 给予准确但有挑战性的信息
- 让调查员有观察和推理的空间
```

#### 特殊场景处理

**场景1: 间谍是候选人之一**
```
说书人额外责任：
- 确保间谍看到你给调查员的信息
- 在"间谍信息"阶段展示

间谍的优势：
✓ 知道调查员会指认他
✓ 可以提前准备伪装
✓ 可以伪装成另一个爪牙

公平性：
✓ 这是间谍的合法优势
✓ 调查员需要通过行为识别
✓ 增加游戏复杂性
```

**场景2: 真爪牙是猩红女郎**
```
特殊考虑：
- 猩红女郎的身份很关键
- 如果她被过早识别和处决
  → 邪恶失去后备方案

说书人平衡：
- 可以选择不那么明显的候选人组合

```

**场景3: 多个信息角色交叉验证**

```
→ Alice 是毒师（调查员信息）
→ 占卜师验证 Alice 不是恶魔（一致！）
→ 完美的信息网络

说书人应该：
✓ 意识到这种组合
✓ 确保信息逻辑自洽（如果都未被干扰）
```

### 关键原则

**1. 信息是资产，需要资产管理**
```
不要浪费：
- 第一天全盘公开 = 浪费观察机会

不要囤积：
- 从不公开 = 浪费信息价值

平衡释放：
- 在最佳时机，最大化价值
```

**2. 验证 > 假设**
```
永远寻求验证：
- 交叉对比其他信息角色
- 观察行为证据
- 寻求第三方确认

不要基于假设行动：
- "我觉得肯定是他" ≠ 证据
```

**3. 灵活 > 固执**
```
保持开放：
- 新信息可能改变判断
- 愿意承认错误
- 快速调整策略

避免认知固化：
- "一定是 X" → 危险信号
- 保留替代假设
```

**4. 团队 > 个人**
```
记住你的目标：
- 帮助善良阵营获胜
- 不是证明自己正确

有时承认错误 > 坚持错误：
- 诚实重建信任
- 错误信息也能转化为线索
```

**5. 观察 > 发言**
```
多听少说（初期）：
- Day 1: 80% 观察，20% 发言
- Day 2: 60% 观察，40% 发言
- Day 3+: 根据情况调整

每句话都有目的：
- 不要随意透露信息
- 每次发言都是战术选择
```

### 常见问题 FAQ

**Q1: 收到"0"信息，但有人宣称爪牙，我该怎么办？**
```
A: 分两步判断：

步骤 1: 确认自己状态
- 我可能是醉酒者吗？
- 我的"0"信息与其他信息冲突吗？
- 有证据支持"0"的真实性吗？

步骤 2: 分析宣称者
- 谁宣称爪牙？
- 他们的行为像爪牙吗？
- 可能是恶魔伪装吗？
- 可能是醉酒者（以为自己是爪牙）吗？

决策：
如果确信"0"准确 → 揭穿宣称者
如果怀疑自己被干扰 → 寻求验证
```

**Q2: 两名候选人都宣称镇民，我怎么判断谁是爪牙？**
```
A: 多维度分析：

行为层面：
- 谁的投票模式更可疑？
- 谁在私下会议中言行不一？
- 谁试图转移话题或控制讨论？

信息层面：
- 谁的角色宣称更合理？
- 其他信息角色支持谁？
- 谁的信息被验证了？

心理层面：
- 谁更紧张？
- 谁的解释过于完美（可能提前准备）？
- 谁的反应更自然？

验证层面：
- 请占卜师验证
- 请共情者观察邻居
- 等待送葬者确认（如果有人死亡）
```

**Q3: 我怎么知道自己是不是醉酒者？**
```
A: 寻找矛盾信号：

强烈信号（你可能是醉酒者）：
- 你的信息与 3+ 个可信来源冲突
- 候选人的行为完全不像爪牙
- 送葬者确认候选人身份与你的信息矛盾
- 收到"0"但明显有多个爪牙存在

弱信号（可能性较低）：
- 信息与 1-2 个来源冲突（可能是他们被干扰）
- 候选人行为模糊（难以判断）

确认方法：
- 如果你死亡，送葬者可以确认你的真实身份
- 在此之前，保持警觉，寻求多重验证
```

**Q4: 间谍是候选人之一，我如何识别？**
```
A: 间谍识别技巧：

关键线索：
- 间谍知道你的信息（说书人告诉他）
- 间谍的反应可能"太贴合"你的指认

测试方法：
1. 观察自发行为（Day 1）
   真爪牙不知道你的信息，会自然行动
   间谍知道，可能"表演"

2. 压力测试
   给候选人压力，观察反应
   间谍的反应可能过度准备

3. 信息泄露测试
   观察谁知道不该知道的信息
   间谍可能无意中泄露

4. 交叉验证
   请其他信息角色验证候选人
   多重证据链锁定间谍
```

**Q5: 我的信息是毒师，但没人表现被下毒，怎么办？**
```
A: 分析可能性：

可能 1: 毒师隐藏得很好
- 毒师可能下毒不明显的角色
- 或者被下毒者没有公开信息
→ 继续观察，寻找其他线索

可能 2: 毒师策略性不下毒
- 避免暴露身份
→ 从行为和投票寻找线索

可能 3: 你被干扰了
- 你的信息可能是错误的
- 实际没有毒师，或者是其他爪牙
→ 寻求验证

行动建议：
- 不要只依赖"谁被下毒"来判断
- 综合行为、投票、发言分析
- 交叉验证你的信息
```

**Q6: 什么时候应该公开调查员身份？**
```
A: 没有固定答案，取决于：

立即公开（Day 1-2）：
✓ 收到"0"且需要建立领导
✓ 候选人之一已明显暴露
✓ 需要快速组织善良阵营

延迟公开（Day 3-4）：
✓ 局势复杂，需要更多观察
✓ 怀疑自己被干扰
✓ 建立私密联盟更有价值

持续隐藏：
✓ 确信自己被干扰，信息无价值
✓ 邪恶势力强大，需要保命
✓ 其他信息角色已建立领导

核心原则：
最大化信息价值，根据局势灵活调整
```

## 🎯 总结

### 核心要点回顾

**调查员的本质**:
```
- 第一夜的强力信息角色
- 专注于识别邪恶阵营的爪牙
- 善良阵营的潜在领导者
- 需要观察、推理、沟通的综合能力
```

**成功的五大支柱**:
```
1. 耐心观察
   - 不要急于公开
   - 收集充分的行为数据

2. 交叉验证
   - 寻找其他信息角色
   - 建立多重证据链

3. 灵活策略
   - 没有固定公式
   - 根据局势动态调整

4. 自我质疑
   - 始终考虑被干扰可能性
   - 寻求信息真实性的证据

5. 团队协作
   - 调查员不是独行者
   - 善良阵营胜利 > 个人正确
```

**避免的五大陷阱**:
```
1. 过早公开 → 失去观察优势
2. 盲目信任 → 忽视被干扰风险
3. 忽视"0" → 浪费强力信息
4. 认知固化 → 死守错误判断
5. 单打独斗 → 缺乏信息验证
```

### 从新手到高手的进阶之路

**新手阶段**: 理解机制，学会基本公开
```
目标：不犯明显错误，发挥基本价值
重点：观察 + 验证 + 适时公开
```

**进阶阶段**: 掌握交叉验证，识别干扰
```
目标：精准判断，建立信息联盟
重点：多重验证 + 行为分析 + 灵活策略
```

**高手阶段**: 心理战术，领导善良阵营
```
目标：掌控全局，最大化胜率
重点：心理博弈 + 复杂推理 + 团队领导
```

### 最后的建议

调查员是一个**看似简单，实则深奥**的角色。

**第一夜你收到信息** → 这只是开始
**真正的挑战在于**:
- 如何解读信息
- 何时使用信息
- 如何验证信息
- 怎样领导团队

**最重要的心态**:
> 记住你是在帮助善良阵营获胜，而不是在证明自己正确。
> 有时承认错误，比坚持错误更有价值。
> 信息的价值不在于拥有，而在于如何使用。

**给新手的鼓励**:
- 不要害怕犯错，每次失败都是学习机会
- 观察高手如何玩，但发展自己的风格
- 调查员的乐趣在于推理过程，享受它

**给高手的挑战**:
- 实验创新策略，突破常规思维
- 教导新手，在教学中深化理解
- 思考说书人视角，理解游戏设计

祝你在托儿河镇的调查中，拨开迷雾，找到真相，守护善良！

### 爪牙快速参考

**毒师 (Poisoner)**:
```
能力: 每晚选择一名玩家下毒，使其能力失效
威胁: 可以第一夜下毒调查员
识别: 观察谁的信息异常，反推被下毒者
```

**间谍 (Spy)**:
```
能力: 看到说书人给善良玩家的所有信息
威胁: 知道你的信息，可以完美伪装
识别: 观察"太贴合"的表现，交叉验证
```

**猩红女郎 (Scarlet Woman)**:
```
能力: 恶魔死亡时，如果存活 5+ 人，她变成恶魔
威胁: 处决恶魔后游戏可能继续
识别: 观察座位关系，可能在恶魔附近
```

**男爵 (Baron)**:
```
能力: 游戏开始时增加 2 个外来者
威胁: 增加混乱，更多外来者宣称
识别: 统计外来者数量，如果过多则有男爵
```
