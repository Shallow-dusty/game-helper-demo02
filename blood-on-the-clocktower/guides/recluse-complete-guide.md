# 隐士（Recluse）深度完整手册

## 📚 角色基础信息

### Role Identity / 角色身份
- **英文名称**: Recluse
- **中文名称**: 隐士
- **阵营 (Team)**: 善良 (Good)
- **类型 (Type)**: 外来者 (Outsider)
- **版本 (Edition)**: Trouble Brewing (暗流涌动)

### Ability Description / 能力描述

**英文原文**:
> "You might register as evil & as a Minion or Demon, even if dead."

**中文翻译**:
> "你可能被视为邪恶阵营，甚至被视为爪牙或恶魔，即使你已死亡。"

---

## ⚙️ 能力机制详解

### Ability Mechanics / 运作机制

#### 被动能力 / Passive Ability
隐士的能力是**完全被动**的，不需要主动触发。

#### 注册机制 / Registration Mechanism

隐士可能影响以下角色的信息获取：

| 信息角色 | 正常结果 | 受隐士影响可能结果 |
|----------|----------|---------------------|
| **共情者** | 检测邻居邪恶数 | 可能将隐士计为邪恶 |
| **占卜师** | 检测恶魔 | 可能将隐士视为恶魔 |
| **调查员** | 检测爪牙 | 可能将隐士视为爪牙 |
| **厨师** | 检测邪恶相邻对数 | 可能将隐士计入邪恶 |

**关键规则**:
- 说书人**有权选择**隐士是否在特定检测中注册为邪恶
- 这不是必然发生的，是**可能**发生的
- 即使隐士死亡，其能力仍然有效

#### 说书人裁量权 / Storyteller Discretion

说书人可以选择：
1. 让隐士正常注册为善良
2. 让隐士注册为邪恶（通用）
3. 让隐士注册为特定爪牙角色
4. 让隐士注册为恶魔

**示例场景**:
```
场景：占卜师查验隐士和另一名玩家
选项A：说书人给"否"（隐士注册为善良）
选项B：说书人给"是"（隐士注册为恶魔）
两者都是合法的！
```

## 🎯 玩法策略

### As the Recluse / 作为隐士

#### 核心挑战

作为隐士，你面临独特的困境：
1. **被误解的风险** - 信息角色可能得到你是邪恶的信息
2. **信任危机** - 很难向他人证明你的善良身份
3. **被处决的风险** - 善良阵营可能因错误信息处决你

#### 开局策略

##### 策略1：早期公开（推荐新手）

**优势**:
- 解释为什么信息可能不准确
- 建立早期信任基础
- 让善良阵营理解混乱来源

**劣势**:
- 邪恶阵营可以利用你的存在制造更多混乱
- 恶魔可能伪装成被隐士影响的受害者

**执行方式**:
```
Day 1早期：
"我是隐士。如果有人的信息显示我是邪恶的，
这可能是因为我的能力。请不要因此怀疑我。"
```

##### 策略2：观察后公开

**优势**:
- 先观察谁的信息可能受到影响
- 可以在关键时刻解释混乱

**劣势**:
- 晚公开可能降低可信度
- 错过解释早期错误信息的时机

**执行方式**:
```
Day 1-2：保持沉默，观察信息角色发言
Day 2-3：当有人的信息指向你时公开身份
```

##### 策略3：保持隐藏

**优势**:
- 避免成为邪恶伪装的借口
- 不干扰正常游戏推理

**劣势**:
- 如果被指认为邪恶，很难辩护
- 可能被善良阵营处决

**适用场景**:
- 你确定说书人没有让你注册为邪恶
- 场上邪恶阵营压力已经很大

### 与信息角色互动 / Interacting with Information Roles

#### 共情者互动

**场景**:
```
共情者坐在你旁边，报告"1个邪恶邻居"
但你知道另一个邻居是真正的善良

解决方案：
1. 公开隐士身份
2. 解释共情者的"1"可能是因为你
3. 帮助共情者重新解读信息
```

#### 占卜师互动

**场景**:
```
占卜师查验你和另一人，得到"是"
占卜师可能认为其中一人是恶魔

解决方案：
1. 公开隐士身份
2. 说明这个"是"可能是因为你
3. 建议占卜师重新查验另一人
```

#### 调查员互动

**场景**:
```
调查员得知你和另一人中有一个爪牙
调查员可能怀疑你们其中之一

解决方案：
1. 公开身份解释
2. 帮助调查员理解真正的爪牙在别处
```

## 🤝 角色关系分析

### Synergies / 配合角色

#### 正面配合

**隐士 + 占卜师**
- 如果占卜师查到你是恶魔，你的公开可以解释
- 帮助占卜师排除假阳性

**隐士 + 共情者**
- 你的存在可以解释共情者的异常数字
- 帮助确认真正的邪恶位置

**隐士 + 调查员**
- 解释为什么调查员可能得到错误信息
- 帮助识别真正的爪牙

### Counter Relationships / 对抗关系

#### 被以下角色影响

**中毒者 (Poisoner)**
- 如果被下毒，你可能**不再**能够注册为邪恶
- 这是一个微妙但重要的机制
- 中毒的隐士能力失效，会正常注册为善良

**间谍 (Spy)**
- 间谍可以利用你的存在制造更多混乱
- 间谍可能声称自己是被隐士影响的受害者

#### 对以下角色有影响

**所有信息角色**
- 你的存在给所有信息角色的判断增加不确定性
- 这对善良阵营整体是**负面**的

## ⚠️ 常见误区

### Mistakes to Avoid / 新手易犯错误

#### 误区1：认为能力必然触发

```
错误理解：
"我是隐士，所以信息角色查我一定会得到邪恶"

正确理解：
隐士能力是"可能"而非"必然"
说书人有完全的裁量权
```

#### 误区2：过度依赖能力

```
错误行为：
"查到我的一定是错的，不用担心"

正确做法：
考虑说书人可能没有让你注册为邪恶
信息可能是准确的
```

#### 误区3：不考虑中毒情况

```
错误假设：
"我是隐士，肯定影响了信息"

注意：
如果你被下毒，能力失效
你会正常注册为善良
这时候信息是准确的
```

#### 误区4：用隐士身份解释所有异常

```
错误做法：
"任何奇怪的信息都是因为我"

正确认识：
隐士只能影响与你相关的信息
无关的异常信息需要另外解释
```

## 🎭 说书人指引

### For Storytellers / 给说书人的建议

#### 何时让隐士注册为邪恶？

**考虑因素**:

1. **游戏平衡**
   - 善良阵营信息太准确？让隐士制造混乱
   - 善良阵营已经很困惑？让隐士正常注册

2. **戏剧性**
   - 关键的占卜师查验？制造悬念
   - 已经确定的善良玩家？保持正常

3. **隐士玩家体验**
   - 让隐士感受到角色的独特性
   - 不要每次都让隐士触发（会很无聊）
   - 也不要从不触发（失去角色意义）

#### 推荐做法

```
建议频率：
- 约 50% 的相关检测让隐士注册为邪恶
- 关键时刻（如占卜师首次查验）可以触发
- 后期游戏可以减少触发频率

注册角色选择：
- 通常注册为"邪恶"即可
- 特殊情况可以注册为特定爪牙
- 注册为恶魔应谨慎使用
```

#### 与其他角色的互动

**隐士 + 间谍**:
```
两者都可以影响信息注册
这会造成极大混乱
说书人需要谨慎平衡
建议：如果有间谍，减少隐士触发频率
```

**隐士 + 醉酒者/中毒**:
```
醉酒/中毒的隐士能力失效
会正常注册为善良
这是重要的平衡机制
```

## 📖 经典案例分析

### Case Study 1: 占卜师的困境

**场景设置**:
- 10人局，隐士是Alice
- 占卜师Bob第一夜查验Alice和Carol
- 说书人让隐士注册为恶魔
- 占卜师得到"是"

**发展过程**:
```
Day 1:
Bob（占卜师）: "我查了Alice和Carol，有恶魔！"
全镇开始怀疑这两人

Day 2:
Alice观察到局势紧张，公开：
"我是隐士，占卜师的结果可能是因为我。"

Bob: "那我需要重新查验Carol..."

Day 3:
Bob查验Carol和Dave，得到"否"
结合隐士信息，确认Alice确实是隐士
Carol被排除嫌疑
```

**学习要点**:
- 隐士适时公开可以修正错误信息
- 占卜师需要重新查验确认
- 团队协作解决信息混乱

### Case Study 2: 共情者的数字之谜

**场景设置**:
- 12人局
- 隐士Dave坐在位置5
- 共情者Eve坐在位置6
- 位置4是普通镇民，位置7是间谍

**信息分析**:
```
Eve（共情者）得到: "2个邪恶邻居"

实际情况：
- 位置4: 镇民（善良）
- 位置5: 隐士Dave（善良，但可能注册邪恶）
- 位置7: 间谍（邪恶）

如果隐士注册为邪恶：
共情者得到2（隐士+间谍）

如果隐士正常注册：
共情者应该得到1（只有间谍）
```

**解决过程**:
```
Eve公开得到"2"
Dave公开隐士身份
分析：如果Dave确实是隐士，那"2"中
有一个是真邪恶，一个是隐士
结论：位置4或位置7有一个邪恶
```

**学习要点**:
- 隐士的存在增加分析复杂度
- 需要考虑多种可能性
- 不能简单相加减

### Case Study 3: 隐士被处决的悲剧

**场景设置**:
- 8人局
- 隐士Frank没有公开身份
- 调查员Grace得知Frank/Harry中有爪牙
- Harry是真正的间谍

**悲剧过程**:
```
Day 2:
Grace: "Frank或Harry是爪牙！"

Day 3:
Frank表现紧张，辩护不力
Harry（间谍）表演冷静，指责Frank

投票：Frank被处决
结果：善良阵营损失一名外来者
Harry继续潜伏
```

**问题分析**:
- Frank没有及时公开隐士身份
- 没有解释为什么可能被误判
- Harry利用情况完美伪装

**正确做法**:
```
当被调查员指出时，Frank应该：
1. 立即公开隐士身份
2. 解释调查员信息可能受影响
3. 要求更多证据确认
4. 将注意力转向验证Harry
```

## 🔧 实战技巧总结

### Quick Reference / 速查表

#### 公开时机决策树

```
是否有信息角色公开了与你相关的"邪恶"信息？
├─ 是 → 考虑公开解释
│    ├─ 信息导致你被强烈怀疑？ → 立即公开
│    └─ 仅是轻微提及？ → 可以观察
│
└─ 否 → 继续观察
     ├─ 你被其他原因怀疑？ → 公开可能无帮助
     └─ 局势稳定？ → 保持隐藏
```

#### 关键原则

1. **你是善良的** - 永远记住你的阵营
2. **能力是双刃剑** - 既可能保护你也可能害你
3. **说书人决定** - 你不能控制是否触发
4. **沟通很重要** - 适时解释可以避免误解
5. **考虑中毒** - 被下毒时能力失效

#### 与队友的配合

```
最佳实践：
1. 私下告诉1-2个可信玩家你的身份
2. 请他们在你被怀疑时帮忙证明
3. 如果信息角色查到你，配合解释
4. 不要过度使用身份作为挡箭牌
```

## 📚 进阶学习资源

### 推荐学习路径

**新手阶段** (1-5局):
- 理解基本能力机制
- 练习适时公开
- 学习解释信息混乱

**进阶阶段** (6-15局):
- 掌握不同公开策略
- 学习与信息角色配合
- 理解中毒对能力的影响

**高手阶段** (16+局):
- 精确判断公开时机
- 利用身份反向推理
- 帮助团队整理复杂信息

### 常见问题 FAQ

**Q1: 我能控制什么时候注册为邪恶吗？**
```
A: 不能。这完全由说书人决定。
   你只能根据情况选择如何应对。
```

**Q2: 如果我被下毒，能力会怎样？**
```
A: 能力失效。你会正常注册为善良。
   这意味着相关信息是准确的。
```

**Q3: 隐士死了还有能力吗？**
```
A: 是的！隐士的能力"即使死亡"仍然有效。
   死亡的隐士仍可能影响占卜师等查验。
```

**Q4: 我应该在什么时候公开身份？**
```
A: 通常在以下情况：
   - 信息角色的信息指向你是邪恶
   - 你即将被处决且需要辩护
   - 团队需要理解信息混乱的来源
```

**Q5: 隐士对善良阵营是好是坏？**
```
A: 整体来说是负面的：
   - 增加信息不确定性
   - 可能导致错误判断
   - 但也可以用来验证说书人倾向
```

## 🎯 总结

### 核心要点回顾

**隐士的本质**:
- 善良阵营的外来者
- 被动能力，无法主动控制
- 可能干扰信息角色的判断

**成功的关键**:
1. 理解能力的不确定性
2. 适时公开避免误解
3. 与信息角色积极沟通
4. 考虑中毒等特殊情况
5. 不要过度依赖身份辩护

**避免的陷阱**:
1. 认为能力必然触发
2. 不考虑中毒情况
3. 过晚公开导致信任危机
4. 用身份解释所有异常

### 最后的建议

隐士是一个**看似简单但充满变数**的角色。

- 你不能控制能力是否触发
- 你能控制的是如何应对
- 沟通和时机是关键

**最重要的一点**:
> 作为隐士，你的目标不是隐藏，而是帮助善良阵营理解信息。
> 当你的存在成为团队的资产而非负担时，你就成功了。

祝你在血染钟楼的世界中，成为最有价值的隐士！
