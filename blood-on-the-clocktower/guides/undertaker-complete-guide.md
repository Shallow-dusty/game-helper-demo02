## 角色概述

### 基本信息
- **阵营**：镇民（Townsfolk）
- **剧本**：暗流涌动（Trouble Brewing）
- **能力描述**：每个白天处决后的夜晚，你会得知被处决玩家的角色
- **能力类型**：被动信息获取型
- **信息可靠性**：高度可靠（受酒鬼/中毒影响）

### 角色定位

Undertaker（送葬者）是暗流涌动剧本中最具战略价值的镇民角色之一。与其他信息型角色（如占卜师、图书管理员）不同，Undertaker 的信息来源于游戏过程中的关键决策——处决。这使得 Undertaker 成为一个**行动驱动型**的信息角色，而非单纯的被动接收者。

送葬者的核心价值在于：
1. **确认性信息**：获得的是被处决玩家的真实角色，而非概率性推测
2. **累积优势**：随着游戏进行，信息量呈指数级增长
3. **战略导向**：能够引导好人阵营的处决策略
4. **验证工具**：可以验证其他玩家的声明真实性

### 能力的独特性

送葬者的能力设计体现了血染钟楼的核心理念——**信息需要代价**。与占卜师每晚自动获取信息不同，送葬者必须通过：
- 说服团队执行处决
- 承受处决好人的风险
- 在存活与获取信息间平衡

这种设计使送葬者成为游戏节奏的调控者，而非单纯的信息提供者。

### 在剧本生态中的位置

在暗流涌动剧本中，送葬者与其他角色形成互补：
- **与占卜师对比**：占卜师提供预测性信息（谁可能是恶魔），送葬者提供确认性信息（谁确实是什么角色）
- **与厨师对比**：厨师提供静态起始信息，送葬者提供动态过程信息
- **与侦查员对比**：侦查员锁定范围，送葬者确认具体身份

送葬者的存在鼓励好人阵营采取**积极处决策略**，而非保守观望，这从根本上改变了游戏的节奏和策略选择。

## 核心机制详解

### 能力触发条件

送葬者的能力看似简单，但包含多个关键触发条件，理解这些细节对于最大化能力价值至关重要。

#### 1. "处决"的精确定义

**能力只在"处决（Execution）"发生时触发**，以下情况会触发：
- 白天提名投票后，获得多数票的玩家被处决
- 处女能力触发导致的处决
- 任何游戏机制明确标记为"处决"的死亡

**以下情况不会触发能力**：
- 恶魔夜间击杀
- 刺客（如果在其他剧本中）的击杀
- 旅行者被驱逐（Exile）
- 士兵被恶魔攻击但不死亡
- 任何非处决方式的死亡

这个区分非常重要，因为它意味着送葬者必须**主动推动白天的处决流程**才能获取信息。

#### 2. 时间节点

- **第一天**：游戏开始时没有处决，送葬者第一晚不会醒来
- **第二天起**：每个白天如果发生处决，当晚送葬者会醒来获得信息
- **醒来时机**：在夜晚流程中，送葬者通常在中后段醒来，在恶魔行动前后

#### 3. 信息内容

送葬者获得的信息是**被处决玩家的真实角色标记（Token）**，这包含几个关键点：

**真实角色优先原则**：
- 如果被处决的是酒鬼（Drunk），送葬者看到的是"酒鬼"标记，而非酒鬼以为的角色
- 如果被处决的是间谍（Spy），送葬者看到的是"间谍"标记，即使间谍登记为镇民
- 如果被处决的是变节者（Recluse）登记为恶魔，送葬者看到的仍是"变节者"标记

**示例场景**：
```
场景 A：酒鬼以为自己是占卜师
- 该玩家声称：我是占卜师，看到 5 号是恶魔
- 被处决后，送葬者得知：酒鬼
- 解读：该玩家的所有信息都是错误的

场景 B：间谍伪装成厨师
- 该玩家声称：我是厨师，看到 2 对邪恶邻居
- 被处决后，送葬者得知：间谍
- 解读：该玩家的信息是恶意误导

场景 C：真占卜师被处决
- 该玩家声称：我是占卜师，看到 3 号是恶魔
- 被处决后，送葬者得知：占卜师
- 解读：该玩家的信息应当被重视
```

#### 4. 酒鬼和中毒影响

送葬者受到游戏中两种主要干扰机制的影响：

**酒鬼（Drunk）**：
- 如果送葬者本身是酒鬼（不知情的情况下），获得的信息可能是错误的
- 说书人可以展示任何角色标记，通常会展示看似合理但误导性的信息
- 酒鬼送葬者的信息不一定完全错误，但无法保证准确性

**中毒（Poisoned）**：
- 投毒者可以在夜间毒害送葬者
- 被毒害的夜晚，送葬者获得的信息是错误的
- 中毒状态可能持续或解除，增加了信息验证的复杂性

**识别自己被干扰的信号**：
- 信息与其他可靠来源严重冲突
- 被处决玩家的行为与送葬者获得的角色完全不符
- 多个夜晚的信息形成内在矛盾

### 能力的价值计算

送葬者的价值不是线性增长的，而是呈现**阶梯式跃升**：

**第 1 次信息（Day 2 处决）**：
- 价值：中等
- 作用：确认 1 个玩家身份
- 战略影响：开始构建信息网络

**第 2 次信息（Day 3 处决）**：
- 价值：高
- 作用：确认 2 个玩家身份，可以开始交叉验证
- 战略影响：能够识别基本的信息矛盾

**第 3 次信息（Day 4 处决）**：
- 价值：极高
- 作用：确认 3 个玩家身份，形成信息链
- 战略影响：可以推导出其他未处决玩家的可能身份

**第 4+ 次信息**：
- 价值：决定性
- 作用：几乎可以确定剩余玩家的身份分布
- 战略影响：直接指向恶魔的位置

这种价值曲线意味着：**送葬者越晚死，价值越高**。这也是为什么恶魔阵营通常会在早期尝试击杀送葬者。

### 信息的二次价值

送葬者的信息不仅在获得时有价值，更在于**回溯验证**：

**回溯验证机制**：
当送葬者确认某个被处决玩家是特定角色后，可以：
1. 回溯该玩家生前的所有发言
2. 验证该玩家提供的信息真实性
3. 重新评估与该玩家相关的所有推理链

**示例**：
```
Day 2: 图书管理员声称在 3 号和 5 号中有一个酒鬼
Day 3: 图书管理员被处决，送葬者确认是真图书管理员
推论: 3 号和 5 号中确实有一个酒鬼，需要重点关注
```

这种回溯能力使送葬者成为**信息质量的认证机构**，所有通过送葬者验证的信息都具有更高的可信度。

### 能力的战略性使用

送葬者不应被动等待处决发生，而应**主动设计处决策略**：

#### 策略 1：优先级处决列表

根据信息价值排序处决目标：
1. **高优先级**：声称关键信息的角色（占卜师、厨师、侦查员）
2. **中优先级**：声称辅助角色或存疑玩家
3. **低优先级**：可信度高的玩家

#### 策略 2：测试性处决

故意处决某些玩家以验证信息网络：
- 处决两个互相印证的玩家之一，验证信息真实性
- 处决边缘玩家，通过排除法缩小恶魔范围

#### 策略 3：保护性揭示

在关键时刻公开送葬者身份，以：
- 阻止团队处决重要好人
- 引导团队处决恶魔
- 在残局提供决定性信息

## 信息验证与交叉印证

送葬者的核心技能不是获取信息，而是**验证和利用信息**。本章节深入探讨如何将送葬者的信息转化为胜利。

### 信息验证的三个层次

#### 第一层：直接验证

**定义**：通过送葬者信息直接确认或否定某个声明。

**应用场景**：
```
场景：玩家 A 声称自己是僧侣，保护过玩家 B
处决玩家 A 后，送葬者得知：间谍
结论：玩家 A 是邪恶阵营，其声明完全不可信
```

**直接验证的价值**：
- 立即清除虚假信息
- 识别恶魔阵营成员
- 重新调整推理方向

**直接验证的局限**：
- 需要实际处决才能验证
- 可能牺牲好人获取信息
- 无法验证未处决的玩家

#### 第二层：间接推导

**定义**：通过已验证信息推导未验证玩家的身份。

**推导逻辑链**：
```
已知信息：
- 送葬者确认玩家 A 是真厨师
- 厨师声称看到 1 对邪恶邻居
- 玩家 A 的邻居是玩家 B 和玩家 C

推导过程：
1. 厨师信息可信
2. B 和 C 中至少有一个是邪恶阵营
3. 如果 B 是镇民（有证据），则 C 必然是邪恶阵营
4. 如果 C 声称是镇民，则 C 可能在撒谎
```

**间接推导的关键**：
- 建立可靠的信息链
- 逐步排除不可能的组合
- 保持逻辑严密性

#### 第三层：概率分析

**定义**：在信息不完整时，通过概率评估最可能的情况。

**概率分析框架**：
```
游戏设定：7 人局
- 镇民：3 人
- 外来者：1 人
- 爪牙：1 人
- 恶魔：1 人

已确认（通过送葬者）：
- 玩家 1：真占卜师（镇民）
- 玩家 2：酒鬼（外来者）

推导：
- 剩余 5 人中：2 镇民、1 爪牙、1 恶魔、1 其他
- 未确认的镇民声明需要谨慎验证
- 爪牙和恶魔会伪装成镇民
```

**概率分析的应用**：
- 评估未验证声明的可信度
- 决定下一个处决目标
- 在信息不足时做出合理决策

### 交叉印证技术

交叉印证是送葬者最强大的工具，通过多个信息源相互验证，建立**不可撼动的信息网络**。

#### 技术 1：双向验证

**原理**：两个玩家的信息互相印证。

**示例场景**：
```
玩家 A 声称：我是图书管理员，看到 3 号和 5 号中有一个酒鬼
玩家 B（5 号）声称：我是酒鬼

处决 A 后，送葬者确认：真图书管理员
推论：B 很可能是真酒鬼，3 号不是酒鬼
```

**双向验证的强度**：
- 高强度：两个信息源都被送葬者验证
- 中强度：一个信息源被验证，另一个逻辑自洽
- 低强度：都未被验证，仅依赖逻辑关联

#### 技术 2：三角验证

**原理**：三个玩家的信息形成闭环，互相支持。

**示例场景**：
```
玩家 A：厨师，看到 1 对邪恶邻居
玩家 B：占卜师，看到 C 不是恶魔
玩家 C：侦查员，看到 D 或 E 是爪牙

已验证（送葬者）：
- A 是真厨师
- B 被处决，确认真占卜师

推导链：
1. A 的信息可信（已验证）
2. B 的信息可信（已验证）
3. C 的声明与 A、B 的信息不矛盾
4. C 很可能是真侦查员
```

**三角验证的优势**：
- 大幅降低邪恶阵营伪造的可能性
- 建立高可信度信息网络
- 为最终推理提供坚实基础

#### 技术 3：排除验证

**原理**：通过排除法缩小可能性范围。

**应用步骤**：
1. 列出所有可能的角色组合
2. 使用送葬者信息排除不可能的组合
3. 分析剩余组合的概率分布
4. 选择最有可能的组合作为工作假设

**示例**：
```
9 人局，已处决 3 人：
- 玩家 1：真占卜师（送葬者确认）
- 玩家 2：爪牙男爵（送葬者确认）
- 玩家 3：酒鬼（送葬者确认）

已知男爵能力：增加 2 个外来者
推导：
- 外来者总数：1（基础）+ 2（男爵）= 3
- 已确认外来者：酒鬼（1 个）
- 剩余 6 人中有 2 个外来者
- 剩余 6 人中：2 镇民、2 外来者、1 恶魔、1 可能的其他爪牙

缩小范围：
- 声称镇民的玩家中，只有 2 个是真的
- 其他声称镇民的要么是外来者，要么是恶魔
```

### 信息质量评估体系

并非所有信息都具有相同价值，送葬者需要建立**信息质量评估体系**：

#### 评估维度 1：确定性

- **绝对确定**：送葬者直接验证的信息
- **高度确定**：多重交叉印证的信息
- **中度确定**：单一来源但逻辑自洽的信息
- **低度确定**：孤立的、未验证的声明
- **不确定**：相互矛盾的信息

#### 评估维度 2：时效性

- **即时信息**：当前夜晚获得的信息
- **近期信息**：1-2 天前获得的信息
- **陈旧信息**：3+ 天前获得的信息（可能受中毒等影响）

#### 评估维度 3：来源可靠性

- **A 级来源**：已被送葬者验证为真的角色
- **B 级来源**：通过交叉印证高度可信的角色
- **C 级来源**：未验证但声明合理的角色
- **D 级来源**：存疑或有矛盾的角色

#### 综合评估示例

```
信息：玩家 X 声称看到玩家 Y 是恶魔

质量评估：
- 确定性：低（未验证）
- 时效性：高（今天获得）
- 来源可靠性：取决于玩家 X 的身份

如果玩家 X 被处决，送葬者确认是真占卜师：
- 确定性：升级为绝对确定
- 来源可靠性：升级为 A 级
- 该信息成为决定性证据
```

### 处理矛盾信息的策略

在实际游戏中，信息矛盾不可避免，送葬者需要系统性方法处理：

#### 策略 1：矛盾分类

**类型 A：直接矛盾**
- 两个玩家声称同一角色
- 一个玩家的信息直接否定另一个玩家

**处理方法**：
1. 优先处决其中一个
2. 用送葬者信息确定谁在撒谎
3. 基于结果调整推理

**类型 B：间接矛盾**
- 两个信息在逻辑上无法同时为真
- 需要推导才能发现的矛盾

**处理方法**：
1. 详细分析逻辑链
2. 寻找其他支持证据
3. 必要时处决验证

**类型 C：可解释矛盾**
- 看似矛盾，但可能由酒鬼、中毒等解释

**处理方法**：
1. 列出所有可能解释
2. 评估每种解释的概率
3. 保留多种可能性，继续收集信息

#### 策略 2：矛盾优先级

并非所有矛盾都需要立即解决：

**高优先级矛盾**：
- 涉及恶魔身份的矛盾
- 影响当前处决决策的矛盾
- 关系到多个玩家可信度的矛盾

**低优先级矛盾**：
- 涉及外来者位置的小矛盾
- 不影响大局的细节矛盾
- 可能由游戏机制解释的矛盾

#### 策略 3：构建信息世界

**世界理论（World Building）**：
为每种可能的情况构建完整的"世界"，检验其一致性。

**步骤**：
1. 假设玩家 X 是恶魔
2. 基于此假设，推导其他玩家的可能身份
3. 检查这个"世界"是否与所有已知信息一致
4. 重复以上步骤，为其他可能的恶魔构建"世界"
5. 比较不同"世界"的合理性，选择最可能的

**示例**：
```
世界 A：玩家 5 是小恶魔
- 玩家 1 必须是间谍（否则信息矛盾）
- 玩家 3 必须是真厨师（送葬者已验证）
- 玩家 7 必须是酒鬼（解释其错误信息）
- 评估：需要 3 个条件同时成立，概率中等

世界 B：玩家 7 是小恶魔
- 玩家 1 是真占卜师（符合送葬者验证）
- 玩家 3 是真厨师（送葬者已验证）
- 玩家 5 是爪牙（解释其误导行为）
- 评估：只需 1 个额外条件，概率更高

结论：世界 B 更可能是真实情况
```

### 信息的时间管理

送葬者必须理解信息的**保质期**：

#### 早期信息（Day 1-2）

**特点**：
- 信息量少
- 不确定性高
- 受干扰可能性大

**使用策略**：
- 建立初步假设
- 保持灵活性
- 避免过度依赖

#### 中期信息（Day 3-4）

**特点**：
- 信息开始形成网络
- 可进行交叉验证
- 关键转折点

**使用策略**：
- 积极交叉验证
- 识别信息矛盾
- 引导关键处决

#### 后期信息（Day 5+）

**特点**：
- 信息高度完整
- 可精确推导
- 决定性阶段

**使用策略**：
- 综合分析所有信息
- 精确定位恶魔
- 指导最终处决

## 开局策略

开局阶段（Day 1-2）对送葬者至关重要，这一阶段的决策将影响整个游戏走向。

### 第一天：身份选择

#### 选项 1：立即公开身份

**优势**：
- 建立信任，获得团队支持
- 引导第一天处决，开始获取信息
- 让其他玩家愿意分享信息

**劣势**：
- 成为恶魔优先击杀目标
- 可能第一晚就被杀
- 邪恶阵营会针对性布局

**适用场景**：
- 团队有僧侣、士兵等保护角色
- 需要快速建立信息优势
- 其他玩家沟通混乱，需要领导

**公开话术示例**：
```
"各位，我是送葬者。我知道这很冒险，但我们需要开始处决来获取信息。
我建议今天处决 [某个存疑玩家]，明天我可以告诉大家他的真实身份。
如果有僧侣，请考虑保护我。"
```

#### 选项 2：部分公开（软揭示）

**策略**：暗示自己是信息型角色，但不明确说明。

**话术示例**：
```
"我建议我们今天必须处决人，这对信息收集很重要。"
"相信我，今天的处决会让明天的形势更清晰。"
```

**优势**：
- 引导处决而不暴露身份
- 降低被击杀风险
- 保留后期揭示的惊喜效果

**劣势**：
- 可能无法获得充分支持
- 其他玩家可能不配合处决策略

#### 选项 3：完全隐藏

**策略**：伪装成其他角色，秘密收集信息。

**伪装选项**：
- **伪装成外来者**（如酒鬼）：降低威胁性
- **伪装成辅助镇民**（如处女）：保持低调
- **不声明角色**：观察态势

**优势**：
- 最大化生存时间
- 收集更多信息
- 在关键时刻揭示产生震撼效果

**劣势**：
- 可能被误处决
- 难以引导处决策略
- 信息获取效率低

### 第一天处决策略

送葬者的第一个信息至关重要，选择正确的处决目标能奠定胜利基础。

#### 目标类型 1：声称强力信息角色的玩家

**典型目标**：
- 声称占卜师并指认恶魔的玩家
- 声称厨师并给出关键数字的玩家
- 声称图书管理员的玩家

**处决理由**：
1. 验证其信息真实性
2. 如果是真的，其信息已经分享，价值已实现
3. 如果是假的，早期识别邪恶阵营成员

**示例对话**：
```
玩家 A："我是占卜师，看到玩家 5 是恶魔。"
送葬者思考：
- 如果 A 是真占卜师，我们已经知道 5 是恶魔
- 如果 A 是假的（间谍或邪恶阵营），我需要尽早知道
- 处决 A，明天我可以验证其身份，然后决定是否处决 5

送葬者发言："我建议处决玩家 A。如果他是真占卜师，我们已经有了
信息。如果他在撒谎，我们需要知道。"
```

#### 目标类型 2：边缘玩家

**定义**：发言少、信息少、存在感低的玩家。

**处决理由**：
1. 这些玩家可能是躲藏的恶魔
2. 处决他们社交成本低
3. 即使是好人，损失也相对较小

**风险**：
- 可能处决重要的沉默型角色（如隐藏的占卜师）
- 信息价值可能较低

#### 目标类型 3：互相矛盾的玩家对

**场景**：
```
玩家 B："我是洗衣妇，看到玩家 C 和 D 中一个是占卜师。"
玩家 C："我是占卜师。"
玩家 D："我也是占卜师。"
```

**处决策略**：
1. 先处决 B（验证洗衣妇身份）
2. 如果 B 是真洗衣妇，明天处决 C 或 D
3. 如果 B 是假的，C 和 D 都可能在撒谎

#### 目标类型 4：战略性测试

**高级策略**：故意处决某个玩家以测试信息网络。

**示例**：
```
已知信息：
- 玩家 X 声称厨师，看到 1 对邪恶邻居
- 玩家 Y 和 Z 是 X 的邻居
- Y 和 Z 都声称是镇民

处决 X 的理由：
1. 如果 X 是真厨师，我确认 Y 或 Z 是邪恶的
2. 如果 X 是假厨师，Y 和 Z 可能都是好人
3. 这个信息对后续推理非常关键
```

### 第一晚后的信息处理

#### 步骤 1：评估信息价值

获得第一个信息后，立即评估：
- **高价值**：确认了关键角色（占卜师、厨师等）
- **中价值**：确认了普通镇民或外来者
- **低价值**：确认了边缘玩家

#### 步骤 2：决定信息分享策略

**完全分享**：
```
"各位，昨天我们处决了玩家 A，我确认他是真占卜师。
这意味着他说的玩家 5 是恶魔的信息是可信的。"
```

**部分分享**：
```
"我可以确认，昨天被处决的玩家没有在撒谎。"
```

**不分享**：
- 将信息作为秘密武器
- 观察其他玩家反应
- 在关键时刻揭示

#### 步骤 3：调整第二天策略

基于第一个信息，规划第二天：
- 如果验证了重要信息，跟进相关线索
- 如果发现了撒谎者，追查其同伙
- 如果信息价值低，调整处决目标选择

### 应对早期压力

送葬者在开局阶段可能面临各种压力：

#### 压力 1：被要求证明身份

**场景**：
```
其他玩家："你说你是送葬者，但你怎么证明？你现在没有任何信息。"
```

**应对策略**：
```
"送葬者第一晚不会获得信息，这是角色机制。如果我们今天处决，
明天我就可以提供证明。这也是为什么我强烈建议今天必须处决。"
```

**高级应对**：
- 详细描述送葬者的能力细节（证明你了解角色）
- 说明送葬者的战略价值
- 请求一天的信任期

#### 压力 2：被怀疑是邪恶阵营

**场景**：
```
质疑者："我觉得你是间谍伪装成送葬者，目的是引导我们错误处决。"
```

**应对策略**：
```
"我理解你的怀疑。但请思考：如果我是间谍，为什么要冒险声称送葬者？
这个角色很容易被验证。只要处决我，就能知道真相。
或者给我一天时间，用信息证明自己。"
```

**逻辑反驳**：
- 间谍伪装成送葬者风险极高
- 真送葬者会主动推动处决（符合角色利益）
- 请求用行动证明

#### 压力 3：被恶魔优先击杀

**第一晚死亡场景**：
- 如果第一晚就被杀，送葬者无法获得任何信息
- 这是最糟糕的结果

**预防措施**：
- 第一天不要过度暴露（除非有保护）
- 如果有僧侣，私下沟通请求保护
- 考虑伪装策略

**如果被杀，遗言策略**：
```
"我是送葬者，被杀说明恶魔害怕我获取信息。
我建议团队继续积极处决，其他信息型角色可以接替我的工作。
重点关注 [你怀疑的玩家]。"
```

### 开局阶段的团队协作

#### 与占卜师协作

**理想场景**：
- 占卜师提供恶魔候选
- 送葬者验证占卜师身份
- 双重确认后处决恶魔

**协作话术**：
```
送葬者："如果你是真占卜师，你愿意被处决来验证你的信息吗？"
占卜师："可以，但我建议先处决我指认的恶魔。"
送葬者："我理解，但验证你的身份能让你之前所有的信息都变得可信。"
```

#### 与厨师/图书管理员协作

**策略**：
- 优先验证这些首夜信息角色
- 他们的信息已经获得，验证后价值最大化
- 形成信息网络的基础

#### 与保护型角色协作

**僧侣**：
```
送葬者私下沟通："我是送葬者，如果你是僧侣，请考虑保护我。
我需要存活到中后期才能发挥最大价值。"
```

**士兵**：
- 士兵无法主动保护，但可以作为诱饵
- 送葬者可以假装不确定士兵身份，降低自己威胁性

## 中期运营

中期阶段（Day 3-5）是送葬者的黄金时期，这时已经积累了 2-3 个信息，可以开始深度分析。

### 信息网络的构建

#### 网络节点理论

将每个被处决的玩家视为**信息节点**，通过节点间的连接构建网络：

**节点类型**：
- **核心节点**：关键信息角色（占卜师、厨师等）
- **连接节点**：提供关联信息的角色（图书管理员、侦查员）
- **验证节点**：用于验证其他信息的角色
- **噪音节点**：信息价值低的角色

**网络构建示例**：
```
Day 2 处决：玩家 1（送葬者确认：厨师）
- 节点信息：看到 2 对邪恶邻居
- 连接：玩家 1 的邻居是玩家 2 和玩家 9

Day 3 处决：玩家 5（送葬者确认：占卜师）
- 节点信息：看到玩家 7 不是恶魔
- 连接：排除玩家 7

Day 4 处决：玩家 3（送葬者确认：间谍）
- 节点信息：玩家 3 生前声称是侦查员
- 连接：玩家 3 的所有信息都是误导

网络分析：
1. 厨师信息可信 → 玩家 2 或 9 是邪恶的
2. 占卜师信息可信 → 玩家 7 不是恶魔
3. 间谍被识别 → 玩家 3 的误导信息失效
4. 综合推导：如果玩家 9 是邪恶的，且不是恶魔（因为占卜师验证 7），
   那么玩家 9 可能是爪牙
```

#### 信息链条的延伸

**单链条**：
```
A 确认 → B 的信息可信 → B 的信息验证 C → C 可能可信
```

**双链条交叉**：
```
A 确认 → B 可信 → B 的信息支持 D
C 确认 → C 的信息也支持 D
结论：D 高度可信
```

**三链条闭环**：
```
A → B → C → 验证 A 的原始信息
形成自证循环，极高可信度
```

### 中期处决策略

#### 策略 1：优先验证高影响力玩家

**目标特征**：
- 提供了关键信息的玩家
- 被多个其他玩家提及的玩家
- 声称看到恶魔的玩家

**处决顺序**：
1. 验证信息提供者
2. 基于验证结果，处决被指认的玩家
3. 继续验证下一个信息提供者

#### 策略 2：系统性排除

**方法**：将存活玩家分组，逐组验证。

**分组原则**：
```
组 1：声称镇民的玩家
组 2：声称外来者的玩家
组 3：未明确声明的玩家

策略：
- 从组 1 开始，处决其中最可疑的
- 每次处决后，重新评估组内其他成员
- 当组 1 清理完毕，转向组 3
```

#### 策略 3：陷阱式处决

**高级技巧**：故意处决某人以测试其他玩家反应。

**示例场景**：
```
已知：玩家 A 和玩家 B 互相印证（声称洗衣妇和对应角色）

陷阱设置：
1. 提议处决玩家 A
2. 观察玩家 B 的反应
   - 如果 B 强烈反对 → 可能真的互相印证
   - 如果 B 不在意 → 可能只是表面配合

3. 执行处决 A
4. 送葬者确认 A 的身份
5. 基于结果判断 B 的可信度
```

### 中毒和酒鬼的识别

中期阶段，必须考虑送葬者自己可能被干扰的情况。

#### 识别中毒的信号

**信号 1：信息突然矛盾**
```
前两天的信息：
- Day 2: 确认玩家 X 是厨师
- Day 3: 确认玩家 Y 是占卜师

Day 4 信息：
- 确认玩家 Z 是... 小恶魔？但这与之前信息矛盾

分析：Day 4 可能被中毒
```

**信号 2：其他玩家的反馈**
```
送葬者："我确认昨天被处决的是间谍。"
其他玩家："但他的行为完全像一个真镇民，这不合理。"

分析：可能送葬者被中毒，或者该玩家是高水平伪装的间谍
```

**应对策略**：
- 公开讨论中毒可能性
- 回溯之前的信息，寻找不一致
- 请求处决投毒者（如果被识别）

#### 识别自己是酒鬼

**关键信号**：
- 多个信息都与团队信息矛盾
- 逻辑上不可能的角色组合
- 其他玩家普遍质疑

**确认方法**：
```
如果怀疑自己是酒鬼：
1. 对比自己的信息和其他可靠信息源
2. 如果存在系统性矛盾 → 可能是酒鬼
3. 公开讨论："我可能是酒鬼，请大家结合其他信息判断。"
```

**酒鬼送葬者的价值**：
- 即使是酒鬼,仍然是镇民
- 可以作为投票力量
- 可以引导处决策略（即使信息错误）

### 中期沟通技巧

#### 技巧 1：渐进式信息披露

**原则**：不要一次性公开所有信息，而是逐步披露。

**好处**：
- 观察其他玩家对每条信息的反应
- 保留信息作为后续验证工具
- 防止邪恶阵营提前针对性布局

**示例**：
```
Day 3 早晨：
送葬者："昨天的处决给了我信息，但我暂时不公开，我想先听听大家的讨论。"

Day 3 中期：
送葬者："好，基于大家的讨论，我可以确认玩家 X 说的是真话。"

Day 3 晚期：
送葬者："具体来说，玩家 X 是真厨师，这意味着..."
```

#### 技巧 2：引导性提问

**目的**：通过提问引导团队思考，而非直接给出答案。

**示例**：
```
直接方式："玩家 5 是间谍,我们应该处决玩家 7。"

引导方式：
"我确认了玩家 5 的身份。大家觉得这对玩家 7 的声明有什么影响？"
→ 让团队自己推导
→ 观察谁能正确推理（可能是聪明的好人）
→ 观察谁故意误导（可能是邪恶阵营）
```

#### 技巧 3：联盟构建

**策略**：与已验证的好人角色形成紧密联盟。

**联盟成员**：
- 已被送葬者验证的真镇民
- 通过交叉印证高度可信的玩家
- 有保护能力的角色

**联盟运作**：
```
私下沟通：
送葬者对已验证的真占卜师："我们两个可以互相支持。你提供预测性信息，
我提供确认性信息。我们可以主导团队决策。"

公开配合：
在讨论中互相支持，形成可信的声音
```

### 应对中期危机

#### 危机 1：被质疑是酒鬼

**应对框架**：
1. **承认可能性**："我确实可能是酒鬼，这是游戏的一部分。"
2. **提供验证方法**："但我们可以通过交叉验证来判断。"
3. **展示逻辑**："如果我是酒鬼，那么 [某些推论] 应该成立，我们可以验证。"

**示例**：
```
质疑者："你的信息和厨师的信息矛盾，你可能是酒鬼。"
送葬者："有可能。但让我们分析：
- 如果我是酒鬼，厨师的信息是对的
- 那意味着玩家 X 和 Y 是邪恶的
- 但玩家 X 已经被 [其他信息] 验证是好人
- 所以更可能的是厨师在撒谎，或者厨师被干扰了
- 我建议我们处决厨师来验证"
```

#### 危机 2：信息过载

**场景**：积累了太多信息，团队无法消化。

**解决方法**：
1. **简化呈现**：只分享最关键的 2-3 条信息
2. **结论先行**：先说结论，再解释推理
3. **视觉化**：用简单图表或列表展示（如果可能）

**示例**：
```
复杂方式：
"我验证了 A 是厨师，B 是占卜师，C 是间谍，基于 A 的信息和 B 的信息，
我们可以推导出 D 可能是..."

简化方式：
"关键信息三条：
1. 厨师的信息是真的
2. 占卜师的信息是真的
3. 玩家 C 是间谍，他的话都是假的
结论：我们应该处决玩家 [X]"
```

#### 危机 3：团队分裂

**场景**：团队对处决目标意见不统一。

**调解策略**：
```
"我理解大家有不同意见。作为送葬者，我的信息支持 [选项 A]。
但如果大家更倾向 [选项 B]，我也可以接受，因为无论处决谁，
我都能获得信息。关键是我们要处决人，而不是陷入僵局。"
```

**高级技巧**：
- 提出折中方案
- 强调团队统一的重要性
- 必要时让步，保持团队士气

## 残局技巧

残局阶段（剩余 3-5 名玩家）是送葬者最关键的时刻,此时的信息积累应该足以精确定位恶魔。

### 残局信息分析

#### 完整信息网络

到残局阶段，送葬者通常已经处决了 4-5 名玩家，结合其他信息源，应该能构建完整图景。

**信息整合框架**：
```
已处决玩家（送葬者确认）：
1. 玩家 A - 真厨师（看到 2 对邪恶邻居）
2. 玩家 B - 真占卜师（看到玩家 C 不是恶魔）
3. 玩家 D - 间谍（伪装成侦查员）
4. 玩家 E - 酒鬼（以为自己是洗衣妇）

剩余存活玩家：
- 玩家 F：声称僧侣
- 玩家 G：声称处女
- 玩家 H：声称士兵
- 送葬者本人

角色配置推导：
- 已确认镇民：厨师、占卜师、送葬者（3 个）
- 已确认外来者：酒鬼（1 个）
- 已确认爪牙：间谍（1 个）
- 剩余：F、G、H 中有 1 个镇民、1 个恶魔、1 个可能的外来者或镇民

分析：
- 如果没有男爵，应该只有 1 个外来者（酒鬼），F、G、H 中 2 个镇民 1 个恶魔
- 如果有男爵（未处决），则可能有额外外来者
```

#### 精确定位恶魔

**方法 1：排除法**
```
基于已知信息：
- 占卜师验证玩家 C 不是恶魔（C 已死或存活但排除）
- 厨师的邻居中有邪恶 → 定位爪牙位置 → 排除
- 剩余玩家中，排除所有不可能是恶魔的

剩余候选：玩家 F 或玩家 H
```

**方法 2：行为分析**
```
玩家 F 的行为模式：
- 一直投票支持处决好人
- 避免发表明确观点
- 与已确认的间谍有过私下交流

玩家 H 的行为模式：
- 积极参与讨论
- 投票记录合理
- 声称士兵但未被恶魔攻击（可疑）

倾向：玩家 F 更可能是恶魔
```

**方法 3：机制推导**
```
如果玩家 H 是真士兵：
- 恶魔不会攻击他（浪费夜晚行动）
- 但游戏中有人夜晚死亡 → 恶魔在攻击其他人
- 这解释了为什么士兵活到现在

如果玩家 H 在撒谎：
- 他可能是恶魔，没人攻击他是因为他就是恶魔
- 或者他是爪牙/外来者，在掩护恶魔

结论：需要进一步验证
```

### 最终处决决策

残局阶段，送葬者的决策直接决定胜负。

#### 决策树模型

```
剩余 3 人（包括送葬者）

情况 A：送葬者确定知道谁是恶魔
→ 策略：说服另一位好人一起处决恶魔
→ 难度：需要证明自己的分析
→ 风险：如果另一位是恶魔的同伙，可能投票失败

情况 B：在两名玩家间不确定
→ 策略：分析投票组合
  - 如果两人都是恶魔/爪牙 → 已经输了
  - 如果一个是恶魔，一个是好人 → 需要说服好人
  - 如果都是好人 → 恶魔已被处决，已经赢了

情况 C：完全不确定
→ 策略：回溯所有信息，重新分析
→ 最后手段：概率判断 + 行为分析
```

#### 投票说服技巧

**技巧 1：完整信息呈现**
```
"让我总结所有信息：
[列出所有送葬者确认的信息]
[展示逻辑推导过程]
[得出结论：玩家 X 是恶魔]

请相信我，如果我们不处决玩家 X，明天就没有机会了。"
```

**技巧 2：情感诉求**
```
"我作为送葬者，整个游戏都在收集信息。我用好人的生命换来了这些确认。
现在是时候利用这些信息了。请投票处决玩家 X。"
```

**技巧 3：反向论证**
```
"如果我在撒谎，如果我是邪恶阵营，那我为什么要 [列出不合理之处]？
唯一合理的解释是我说的是真的，玩家 X 就是恶魔。"
```

### 残局特殊场景

#### 场景 1：送葬者被质疑是酒鬼

**应对**：
```
"即使我是酒鬼，请看我的逻辑推理：
[展示基于其他信息源的推理]
这些不依赖于我的送葬者信息，而是基于大家都知道的事实。
所以即使我是酒鬼，这个结论仍然成立。"
```

#### 场景 2：恶魔伪装成送葬者

**识别方法**：
```
真送葬者可以回顾每一次处决，详细描述获得的信息
假送葬者可能：
- 信息含糊
- 无法解释信息的推导过程
- 信息与已知事实矛盾
```

**反击策略**：
```
"你说你也是送葬者？好，那请回答：
- Day 2 我们处决了谁？你得到什么信息？
- 那个信息如何与 [某事实] 相符？
- 基于那个信息，你为什么提议 [某处决]？

如果你答不上来，证明你在撒谎。"
```

#### 场景 3：平局投票

**机制**：如果投票平局，没有人被处决。

**策略**：
```
作为送葬者，避免平局至关重要，因为：
- 平局 = 没有处决 = 送葬者不获得信息
- 恶魔又有一晚击杀机会
- 好人数量进一步减少

因此：
- 积极打破僵局
- 必要时妥协，接受次优目标
- 确保至少有处决发生
```

### 胜利确认

#### 如何确认已经赢了

**标准 1：恶魔已被处决**
```
如果送葬者确认某次处决的是小恶魔（或其他恶魔）：
→ 游戏立即结束，好人胜利
→ 送葬者的使命完成
```

**标准 2：逻辑锁定**
```
剩余 3 人，送葬者通过信息确认：
- 另外两人都是好人
→ 恶魔已在之前被处决
→ 好人胜利（即使不知道恶魔是哪次处决的）
```

#### 如何确认已经输了

**标准 1：好人数量 ≤ 邪恶数量**
```
如果在白天阶段：
- 存活玩家：2 个好人 + 2 个邪恶
→ 邪恶阵营可以控制投票
→ 好人无法处决恶魔
→ 游戏结束，邪恶胜利
```

**标准 2：关键角色全灭**
```
如果：
- 所有信息型角色都被杀
- 包括送葬者本人
- 剩余玩家无法确定恶魔身份
→ 好人很可能输掉
```

### 送葬者的遗言

如果送葬者在残局被击杀或处决，遗言至关重要。

**遗言模板**：
```
"我是真送葬者，这是我确认的所有信息：
1. [Day X] 玩家 A - 角色 A
2. [Day Y] 玩家 B - 角色 B
3. [Day Z] 玩家 C - 角色 C

基于这些信息，我的推论是：
- 恶魔是玩家 [X] 或玩家 [Y]
- 玩家 [Z] 是可信的好人

我建议明天处决玩家 [X]。

祝好人阵营胜利。"
```

**遗言策略**：
- **清晰**：使用简单语言
- **完整**：包含所有关键信息
- **可执行**：给出明确建议
- **鼓舞**：维持团队士气

## 与其他角色的协作

送葬者不是孤立角色，与其他角色的协作能极大提升团队胜率。

### 与信息型镇民协作

#### 与占卜师（Fortune Teller）协作

**协作模式 1：验证链**
```
流程：
1. 占卜师指认玩家 X 是恶魔
2. 团队处决占卜师（听起来反直觉，但有战略价值）
3. 送葬者确认占卜师身份
4. 如果是真占卜师 → 其信息可信 → 处决玩家 X
5. 如果是假占卜师 → 其信息不可信 → 重新评估
```

**协作模式 2：双重确认**
```
流程：
1. 占卜师每晚提供预测
2. 送葬者通过处决验证预测
3. 形成预测-验证循环
4. 快速锁定恶魔
```

**最佳实践**：
```
Day 2：
占卜师："我看到玩家 5 是恶魔。"
送葬者："我建议先处决你（占卜师）来验证。"
占卜师："理解，但如果我是真的，浪费了信息来源。"
送葬者："但如果验证你是真的，你的信息就变成确认事实，价值更高。"

Day 3：
送葬者确认占卜师是真的
团队立即处决玩家 5
→ 如果玩家 5 是恶魔，游戏结束，好人胜利
→ 如果不是，说明占卜师被中毒或看错了，继续分析
```

#### 与厨师（Chef）协作

**协作价值**：
- 厨师提供邪恶邻居数量
- 送葬者验证厨师身份
- 确认后，厨师信息成为定位爪牙/恶魔的关键

**示例**：
```
厨师："我看到 2 对邪恶邻居。"
这意味着某些相邻的玩家中有 2 对是邪恶阵营

送葬者策略：
1. Day 2 处决厨师本人，确认身份
2. 确认是真厨师后，分析座位顺序
3. 结合其他信息，锁定邪恶邻居对
4. 逐一处决验证
```

**高级技巧**：
```
如果厨师说看到 0 对邪恶邻居：
→ 所有邪恶阵营不相邻
→ 这是强力信息，极大缩小搜索范围

如果厨师说看到 2 对邪恶邻居：
→ 在小规模游戏中（7-9 人），这意味着邪恶阵营很集中
→ 可能有男爵（增加外来者，导致更多邪恶）
```

#### 与图书管理员（Librarian）协作

**协作模式**：
```
图书管理员："我看到玩家 3 和玩家 6 中有一个是酒鬼。"

送葬者策略：
1. 处决图书管理员，验证身份
2. 如果是真图书管理员，处决玩家 3 或 6
3. 送葬者确认被处决者身份：
   - 如果是酒鬼 → 另一个不是酒鬼
   - 如果不是酒鬼 → 另一个是酒鬼，或图书管理员被干扰
```

**特殊价值**：
- 识别酒鬼对送葬者极其重要
- 酒鬼的错误信息会误导团队
- 早期识别酒鬼可以忽略其误导

#### 与侦查员（Investigator）协作

**协作模式**：
```
侦查员："玩家 4 或玩家 7 中有一个是男爵。"

送葬者策略：
1. 验证侦查员身份（处决）
2. 确认是真侦查员后，处决玩家 4 和 7 之一
3. 通过送葬者信息确认是否有男爵
4. 如果有男爵，调整外来者数量预期
```

**男爵的重要性**：
- 男爵会增加 2 个外来者
- 这改变了角色分布计算
- 必须准确识别以正确推理

#### 与洗衣妇（Washerwoman）/ 图书管理员（Librarian）/ 侦查员（Investigator）协作

这三个角色都提供"N 选 1"类型的信息，协作模式类似：

**通用策略**：
```
1. 这些角色通常在 Day 1 就分享信息
2. 送葬者评估信息价值
3. 在 Day 2-3 处决这些角色以验证
4. 验证后，其信息成为确认事实
5. 基于确认事实，处决被指认的玩家
```

### 与保护型镇民协作

#### 与僧侣（Monk）协作

**理想协作**：
```
Day 1 私下沟通：
送葬者找到疑似僧侣的玩家："如果你是僧侣，请保护我。
我需要存活到中后期才能发挥最大价值。"

僧侣的保护策略：
- 早期保护送葬者（高价值目标）
- 中期保护已验证的关键镇民
- 后期保护自己或其他关键角色
```

**挑战**：
- 僧侣无法保护同一人连续两晚
- 送葬者需要假设不会每晚都被保护
- 必要时仍需保持低调

#### 与士兵（Soldier）协作

**士兵的特殊性**：
- 士兵免疫恶魔攻击
- 但恶魔不知道谁是士兵（除非有间谍）

**协作策略**：
```
送葬者可以：
1. 公开质疑谁是士兵
2. 引导恶魔攻击疑似士兵的玩家（保护自己）
3. 通过处决验证士兵身份

士兵可以：
1. 适度暴露，吸引恶魔攻击
2. 保护送葬者免受怀疑（转移注意力）
```

**验证士兵的价值**：
```
如果送葬者确认某玩家是真士兵：
→ 该玩家的投票完全可信
→ 可以作为最终决战的盟友
→ 残局时，士兵 + 送葬者 + 1 可信玩家 = 稳定胜利
```

### 与功能型镇民协作

#### 与市长（Mayor）协作

**市长能力**：如果只剩 3 名玩家且没有处决，好人胜利。

**协作策略**：
```
残局场景（3 名玩家存活）：
- 送葬者
- 市长
- 疑似恶魔

策略 A（处决）：
- 送葬者说服市长一起处决疑似恶魔
- 如果成功，游戏结束，好人胜利

策略 B（不处决）：
- 如果不确定谁是恶魔
- 可以选择不处决，触发市长能力
- 好人自动胜利

挑战：
- 必须确认市长是真的（可能之前被送葬者验证）
- 如果"市长"是恶魔在伪装，策略 B 会导致失败
```

#### 与处女（Virgin）协作

**处女能力**：如果被镇民提名，该镇民立即被处决。

**协作陷阱**：
```
送葬者不应提名声称处女的玩家，除非：
1. 确定自己不是酒鬼
2. 想测试对方是否真处女
3. 愿意牺牲自己获取信息

如果送葬者提名处女并被处决：
→ 送葬者死亡，但团队确认处女是真的
→ 这是一次信息交换，可能值得（残局时）
```

#### 与送终人（Undertaker）协作

等等，这是送葬者自己的章节，不存在"另一个送葬者"。

（注：在暗流涌动剧本中，每个镇民角色只出现一次）

### 与外来者的关系

#### 与酒鬼（Drunk）的关系

**关键点**：
- 酒鬼不知道自己是酒鬼
- 酒鬼以为自己是镇民
- 酒鬼的信息是错误的

**送葬者的作用**：
```
如果送葬者处决并确认某玩家是酒鬼：
1. 该玩家之前的所有"信息"都是错误的
2. 团队必须抛弃基于那些信息的推理
3. 重新构建逻辑框架

示例：
玩家 A 一直声称是占卜师，提供了 3 晚信息
Day 4 被处决，送葬者确认：酒鬼
→ 玩家 A 的 3 晚"占卜"信息全部作废
→ 基于那些信息被怀疑的玩家可能是清白的
→ 游戏局势彻底改变
```

**挑战**：
- 识别酒鬼是送葬者的关键任务之一
- 酒鬼的误导可能导致好人自相残杀
- 必须尽早识别并纠正

#### 与变节者（Recluse）的关系

**变节者能力**：可能被登记为邪恶角色（包括恶魔或爪牙）。

**对送葬者的影响**：
```
送葬者处决某玩家，确认身份：变节者

其他角色可能看到的信息：
- 占卜师可能看到变节者"是恶魔"
- 厨师可能把变节者算作"邪恶邻居"
- 侦查员可能看到变节者"是爪牙"

送葬者的价值：
→ 送葬者看到的是真实角色（变节者）
→ 可以纠正其他角色的误判
→ 避免团队误杀好人
```

**策略**：
```
如果占卜师声称玩家 X 是恶魔，但送葬者确认玩家 X 是变节者：
→ 向团队解释："玩家 X 是变节者，不是恶魔。占卜师的信息因为
   变节者的能力而产生误判。我们应该寻找真正的恶魔。"
```

#### 与圣徒（Saint）的关系

**圣徒能力**：如果圣徒被处决，游戏结束，邪恶阵营胜利。

**送葬者的噩梦场景**：
```
团队处决某玩家
送葬者获得信息：圣徒
→ 游戏立即结束，好人失败
→ 送葬者甚至没有机会分享这个信息
```

**预防策略**：
```
1. 如果有玩家声称圣徒，极其谨慎对待
2. 优先处决其他目标
3. 只有在绝对确定的情况下才处决疑似圣徒
4. 在残局时，如果剩余玩家中可能有圣徒，考虑不处决（依赖市长能力等）
```

**邪恶阵营的伪装**：
- 恶魔可能伪装成圣徒，避免被处决
- 送葬者需要通过其他方式识别真假圣徒
- 可以依赖圣徒的行为模式（真圣徒通常低调,假圣徒可能过度强调）

#### 与管家（Butler）的关系

**管家能力**：只能投与主人相同的票。

**对送葬者的影响**：
```
如果送葬者确认某玩家是管家：
→ 该玩家的投票不完全自主
→ 需要识别管家的"主人"
→ 管家的投票模式可能揭示主人身份

策略应用：
1. 观察管家的投票，推断主人是谁
2. 如果主人是邪恶的，管家会被迫投邪恶的票
3. 如果主人是好人，管家的投票可信
```

### 对抗邪恶阵营

#### 对抗间谍（Spy）

**间谍能力**：
- 看到魔法书（知道所有角色）
- 可能被登记为好人角色
- 知道所有信息（包括占卜师的选择等）

**送葬者的优势**：
```
送葬者是间谍的克星：
1. 送葬者处决间谍后，能看到真实身份"间谍"
2. 即使间谍登记为镇民，送葬者仍看到"间谍"
3. 这是少数能穿透间谍伪装的能力之一
```

**识别间谍的策略**：
```
信号 1：信息过于准确
- 某玩家声称的"推理"准确度异常高
- 可能是间谍利用魔法书信息

信号 2：行为矛盾
- 声称某角色，但行为不符合该角色利益
- 例如：声称占卜师，但不积极推动处决"恶魔"

信号 3：过度配合
- 间谍可能伪装成好人，过度配合团队
- 目的是建立信任，在关键时刻误导

处决验证：
- 送葬者优先处决高度可疑的玩家
- 一旦确认是间谍，其所有信息作废
```

#### 对抗投毒者（Poisoner）

**投毒者能力**：每晚可以毒害一名玩家，该玩家的能力失效。

**对送葬者的威胁**：
```
如果送葬者被毒害：
→ 当晚获得的信息是错误的
→ 可能误导整个团队
→ 难以识别何时被毒害
```

**应对策略**：
```
策略 1：信息交叉验证
- 不单独依赖某一晚的信息
- 与其他信息源对比
- 如果发现矛盾，考虑中毒可能

策略 2：识别投毒模式
- 投毒者可能在关键夜晚毒害送葬者
- 例如：处决重要玩家的那晚
- 如果该晚信息异常，可能是被毒害

策略 3：引导处决投毒者
- 通过其他信息源识别投毒者
- 优先处决投毒者，消除干扰
- 一旦投毒者被处决，后续信息可靠
```

**投毒者的处决价值**：
```
如果送葬者确认某玩家是投毒者：
→ 极高价值
→ 消除信息污染源
→ 后续所有角色能力恢复正常
→ 好人阵营信息优势大增
```

#### 对抗男爵（Baron）

**男爵能力**：增加 2 个外来者，减少 2 个镇民。

**对送葬者的影响**：
```
如果有男爵：
- 外来者数量异常（多 2 个）
- 镇民数量减少
- 角色分布计算改变

送葬者的应对：
1. 通过处决确认外来者数量
2. 如果外来者多于预期，推断有男爵
3. 调整推理框架，适应新的角色分布
```

**示例**：
```
7 人局正常配置：
- 3 镇民 + 1 外来者 + 1 爪牙 + 1 恶魔

如果有男爵：
- 1 镇民 + 3 外来者 + 2 爪牙 + 1 恶魔
  （男爵本身算爪牙，所以爪牙从 1 变成 2）

送葬者处决后确认：
- 玩家 A：酒鬼（外来者）
- 玩家 B：变节者（外来者）
- 玩家 C：圣徒（外来者）

推论：有男爵！调整后续策略
```

#### 对抗小恶魔（Imp）

**小恶魔能力**：每晚杀死一名玩家；如果自杀，一名爪牙变成小恶魔。

**送葬者的终极目标**：处决小恶魔。

**挑战**：
```
挑战 1：小恶魔转移
- 如果小恶魔自杀（星传），爪牙变成新恶魔
- 送葬者可能已经处决了原恶魔，但不知道已经换人
- 必须识别新恶魔

挑战 2：伪装
- 小恶魔会伪装成镇民
- 可能声称任何角色
- 需要通过行为和信息矛盾识别

挑战 3：时间压力
- 每晚小恶魔杀人，好人数量减少
- 必须在好人数量降到危险线之前找到恶魔
```

**识别小恶魔的方法**：
```
方法 1：排除法
- 通过送葬者信息，确认已处决的玩家中谁不是恶魔
- 缩小恶魔候选范围
- 最终锁定

方法 2：行为分析
- 小恶魔会避免被怀疑
- 可能过度配合团队
- 但在关键投票时可能露出马脚

方法 3：信息矛盾
- 如果某玩家的声明与送葬者确认的信息矛盾
- 该玩家可能是恶魔（或其他邪恶角色）

方法 4：占卜师配合
- 如果占卜师指认恶魔，且占卜师被送葬者验证为真
- 直接处决被指认的玩家
```

## 高阶策略与陷阱

本章节探讨送葬者的高级玩法，包括心理战术、信息操控和常见陷阱。

### 高阶策略

#### 策略 1：信息延迟释放

**原理**：不立即分享获得的信息，而是战略性地选择时机。

**应用场景**：
```
Day 3 早晨：
送葬者获得信息：昨天处决的是间谍

选项 A（立即公开）：
"昨天被处决的玩家是间谍！"
→ 团队立即知道，可以调整策略
→ 但邪恶阵营也知道，会针对性应对

选项 B（延迟公开）：
送葬者保密，观察其他玩家反应
→ 看谁会为间谍辩护（可能是同伙）
→ 看谁的声明与间谍信息相关（测试真实性）
→ 在最佳时机揭示，达成最大冲击
```

**优势**：
- 观察信息不对称下的玩家行为
- 识别隐藏的邪恶阵营成员
- 在关键时刻扭转局势

**风险**：
- 如果送葬者被杀,信息可能永久丢失
- 团队可能因缺乏信息做出错误决策
- 可能被质疑为什么隐瞒信息

#### 策略 2：虚假信息测试

**原理**：故意释放虚假信息，观察谁会利用或反驳。

**高级技巧**：
```
送葬者公开声称：
"昨天被处决的是真厨师，他的信息可信。"

实际真相：
送葬者知道是间谍，但故意说错

观察：
- 真正的好人可能会质疑（基于其他信息）
- 邪恶阵营可能会接受（因为有利于他们）
- 间谍的同伙可能会过度支持

数小时后揭示真相：
"抱歉，我之前在测试。真相是那是间谍，而玩家 X 的反应证明了他的立场。"
```

**注意**：
- 这是极高风险策略
- 可能破坏团队信任
- 只在有经验的高级局使用
- 必须最终揭示真相

#### 策略 3：牺牲式处决

**原理**：故意处决好人，以验证关键信息。

**场景**：
```
两名玩家互相印证：
- 玩家 A 声称洗衣妇，看到 B 和 C 中一个是占卜师
- 玩家 B 声称占卜师

问题：他们是真的互相印证，还是邪恶阵营配合演戏？

送葬者策略：
Day 2: 处决玩家 A（洗衣妇）
→ 送葬者确认：真洗衣妇
→ 推论：玩家 B 很可能是真占卜师

Day 3: 基于玩家 B（占卜师）的信息行动
→ 高置信度，因为洗衣妇已验证

代价：
→ 失去了一个真镇民（洗衣妇）
→ 但换来了确认的信息网络
→ 长远来看，价值可能更高
```

**计算**：
```
牺牲一个好人的价值 vs. 确认信息的价值

如果信息能够直接指向恶魔 → 值得
如果信息只是边缘辅助 → 不值得
```

#### 策略 4：恶魔诱饵

**原理**：伪装成高价值目标，吸引恶魔攻击，保护其他关键角色。

**执行**：
```
Day 1:
送葬者公开身份，成为明显目标

私下：
送葬者与僧侣沟通，请求保护

结果：
- 恶魔可能攻击送葬者（被僧侣保护，无效）
- 或者恶魔攻击其他人（送葬者存活，获得更多信息）

双赢：
- 如果恶魔攻击送葬者 → 浪费了恶魔的行动
- 如果恶魔不攻击送葬者 → 送葬者继续收集信息
```

**风险**：
- 僧侣可能不存在或无法每晚保护
- 恶魔可能识破诱饵
- 如果失败，送葬者早期死亡

#### 策略 5：残局信息炸弹

**原理**：在残局突然揭示身份和所有信息，形成决定性冲击。

**场景**：
```
游戏进行到 Day 5，剩余 4 名玩家
送葬者一直隐藏身份，伪装成其他角色

此时送葬者揭示：
"我一直是送葬者。这是我确认的所有信息：
[列出 4-5 个已处决玩家的角色]
基于这些信息，恶魔只能是 [玩家 X]。"

冲击效果：
- 团队突然获得大量确认信息
- 邪恶阵营措手不及，难以反驳
- 基于确凿证据，迅速处决恶魔
```

**适用条件**：
- 送葬者确信自己不是酒鬼
- 已经收集了足够信息
- 残局时机成熟，可以一击制胜

### 心理战术

#### 战术 1：信任建立

**原理**：通过一致的行为和准确的信息建立信任。

**步骤**：
```
Step 1：早期小额度的准确预测
"我认为玩家 X 可疑，建议处决。"
→ 处决后，送葬者私下验证，如果准确，公开

Step 2：逐步揭示信息
每天分享一条确认的信息
→ 建立可靠性声誉

Step 3：关键时刻的大胆断言
基于积累的信任，在关键投票时：
"相信我，我们必须处决玩家 Y。"
→ 团队因为之前的信任而支持
```

#### 战术 2：压力测试

**原理**：对可疑玩家施加压力，观察其反应。

**技巧**：
```
送葬者对可疑玩家：
"我有信息显示你可能在撒谎。你想解释吗？"

观察：
- 真好人：通常会冷静解释，提供细节
- 邪恶阵营：可能慌乱、过度辩护、或攻击送葬者

跟进：
"好的，我给你一个机会证明。如果你是真的 [角色]，那么
[某个推论] 应该成立。我们来验证。"
```

#### 战术 3：分化邪恶阵营

**原理**：让邪恶阵营成员互相怀疑，削弱协作。

**方法**：
```
送葬者在公开场合：
"我确认昨天被处决的是爪牙。这意味着剩余的邪恶阵营中，
恶魔可能开始担心爪牙会暴露他。"

效果：
- 如果还有爪牙存活，可能担心被恶魔抛弃
- 恶魔可能担心爪牙失误
- 邪恶阵营内部信任降低，配合效率下降
```

#### 战术 4：声东击西

**原理**：公开关注某个目标,实际目标是另一个。

**执行**：
```
公开：
送葬者："我认为玩家 5 非常可疑，我们应该重点关注。"

实际：
送葬者私下已经确定玩家 7 是恶魔

目的：
- 让恶魔/爪牙以为送葬者没发现他们
- 降低恶魔对送葬者的威胁感知
- 给送葬者更多时间收集信息

最终：
在关键时刻突然转向："其实，我一直在确认，恶魔是玩家 7。"
```

### 常见陷阱与应对

#### 陷阱 1：过度依赖送葬者信息

**问题**：团队完全依赖送葬者，忽略其他信息源。

**后果**：
- 如果送葬者是酒鬼，整个团队误入歧途
- 如果送葬者被长期中毒，信息全部错误
- 团队失去独立思考能力

**应对**：
```
送葬者应该强调：
"我的信息很重要，但不是唯一的。请结合其他角色的信息，
共同推理。不要盲目相信任何单一来源。"

鼓励：
- 其他玩家分享信息
- 交叉验证
- 集体讨论和质疑
```

#### 陷阱 2：被邪恶阵营误导处决

**问题**：邪恶阵营建议处决某人，送葬者跟随，结果处决了关键好人。

**场景**：
```
间谍："我觉得玩家 X 很可疑，应该处决。"
送葬者："好，我也同意。"

结果：
玩家 X 被处决，送葬者确认：真占卜师
→ 好人失去重要信息源
→ 间谍成功误导
```

**应对**：
```
送葬者应该：
1. 独立评估每个处决建议
2. 考虑谁在推动处决，动机是什么
3. 如果不确定，要求更多讨论
4. 优先处决自己选择的目标，而非他人建议
```

#### 陷阱 3：早期暴露导致速死

**问题**：Day 1 公开送葬者身份，Night 1 被恶魔击杀。

**后果**：
- 送葬者没有获得任何信息
- 团队失去关键角色
- 游戏难度大增

**应对**：
```
评估风险后再决定是否公开：
- 有僧侣保护？→ 可以公开
- 团队需要领导？→ 可以公开
- 没有保护，团队尚有秩序？→ 暂时隐藏
```

#### 陷阱 4：忽略自己可能是酒鬼

**问题**：送葬者坚信自己的信息，但实际是酒鬼，误导全队。

**识别信号**：
```
- 多个信息与其他可靠来源矛盾
- 推理出的角色分布不合理（如 5 个镇民在 7 人局）
- 其他玩家普遍质疑

如果多个信号出现：
→ 必须认真考虑自己是酒鬼的可能性
→ 公开讨论，让团队决定是否信任
→ 保持谦逊，不强行推动决策
```

#### 陷阱 5：信息过载导致分析瘫痪

**问题**：收集了大量信息，但无法综合分析，错过关键时机。

**症状**：
```
送葬者：
"我确认了 5 个角色，但我还需要更多信息才能确定恶魔..."

团队：
"我们已经没有时间了！好人数量不够了！"

结果：
→ 过度追求完美信息
→ 错过处决恶魔的机会
→ 好人人数降到危险线以下
```

**应对**：
```
认识到：
- 完美信息是不可能的
- 必须在信息不完整时做出决策
- 概率判断优于无限等待

实践：
- 设定决策时间点（如"Day 4 必须锁定恶魔候选"）
- 基于当前最佳信息行动
- 接受可能的错误，总比不行动好
```

#### 陷阱 6：被间谍的虚假信息链迷惑

**问题**：间谍知道所有角色，可以编造完美的谎言。

**场景**：
```
间谍伪装成厨师："我看到 2 对邪恶邻居。"
→ 间谍知道真实座位，可以编造可信的数字

送葬者处决其他目标，未验证"厨师"
→ 基于虚假信息推理，整个框架错误
```

**应对**：
```
优先验证关键信息提供者：
- 厨师、占卜师等强力角色
- 即使他们看起来可信，也要处决验证
- 不要让任何未验证的信息成为推理基础
```

### 元游戏考量

#### 说书人倾向

**说书人的作用**：在血染钟楼中，说书人（Storyteller）不是中立的，而是为了游戏平衡和娱乐性做出选择。

**对送葬者的影响**：
```
如果送葬者过于强势：
- 说书人可能增加中毒频率
- 或者让酒鬼以为自己是送葬者
- 平衡好人阵营的信息优势

如果好人劣势：
- 说书人可能减少干扰
- 让送葬者获得关键信息
- 帮助好人扳回局势
```

**应对**：
- 理解说书人的平衡作用
- 不要假设信息总是可靠的
- 也不要假设总是被干扰
- 根据游戏状态调整信任度

#### 玩家群体风格

**不同玩家群体的特点**：
```
新手局：
- 信息较直白
- 邪恶阵营伪装不深
- 送葬者信息价值极高

老手局：
- 复杂的心理战
- 多层欺骗和反欺骗
- 送葬者需要更深入的分析

混合局：
- 新手可能无意中帮助或伤害好人
- 老手可能利用新手
- 送葬者需要平衡不同玩家的可信度
```

**调整策略**：
- 观察玩家经验水平
- 对新手更直接
- 对老手更谨慎
- 根据群体风格调整沟通方式

## 总结

### 送葬者的核心价值

Undertaker（送葬者）是暗流涌动剧本中最具战略深度的角色之一。其核心价值在于：

1. **确认性信息**：提供游戏中最可靠的信息来源
2. **累积优势**：随着游戏进行，信息价值呈指数增长
3. **团队领导**：自然成为好人阵营的信息中枢和决策核心
4. **验证工具**：能够验证其他所有信息源的真实性

### 成功送葬者的关键技能

1. **信息管理**：知道何时收集、何时分享、何时隐藏信息
2. **逻辑推理**：构建信息网络，交叉验证，系统性排除
3. **沟通说服**：引导团队处决策略，建立信任，关键时刻说服
4. **风险评估**：平衡信息获取与生存，决定处决优先级
5. **心理洞察**：识别邪恶阵营的伪装，观察行为模式

### 最终建议

**给新手送葬者**：
- 专注于基础：处决 → 获取信息 → 分享信息 → 引导下一次处决
- 不要过度复杂化，信任你的信息
- 积极与团队沟通，建立合作关系
- 如果不确定，优先保守策略

**给进阶送葬者**：
- 实验高级策略：延迟释放、虚假测试、牺牲式处决
- 深入心理战：压力测试、分化邪恶、声东击西
- 理解元游戏：说书人倾向、玩家风格、剧本生态
- 追求最优解，但接受不完美

**给所有送葬者**：
- 记住这是社交游戏，享受过程
- 信息是工具，不是目的
- 团队合作优于个人英雄主义
- 每次失败都是学习机会

### 持续进步

成为优秀的送葬者需要：
- **实践经验**：多玩，观察不同场景
- **复盘反思**：游戏后分析哪些决策正确，哪些可以改进
- **学习他人**：观看高水平玩家的游戏，学习技巧
- **理论研究**：深入理解游戏机制、概率、心理学

送葬者不仅仅是一个角色，更是一种思维方式——**系统性地收集信息，严谨地验证真实，果断地采取行动**。这种能力不仅适用于血染钟楼，也能提升现实生活中的分析和决策能力。

祝所有送葬者玩家在游戏中获得乐趣，在推理中找到满足，在胜利中感受成就！

## 附录：快速参考

### 送葬者能力速查

- **触发**：每个白天有处决发生
- **时间**：当晚醒来
- **信息**：被处决玩家的真实角色
- **限制**：只限处决，不包括其他死亡方式
- **干扰**：酒鬼（系统性错误）、中毒（特定夜晚错误）

### 处决优先级模板

**高优先级**：
1. 声称强力信息角色的玩家（占卜师、厨师等）
2. 互相矛盾的玩家对
3. 高度可疑的行为异常者

**中优先级**：
4. 边缘玩家（低参与度）
5. 未验证的辅助角色
6. 战略性测试目标

**低优先级**：
7. 已被其他方式验证的可信玩家
8. 声称圣徒的玩家（极度谨慎）
9. 明显的好人（保留到后期）

### 信息质量评分卡

| 维度 | A 级 | B 级 | C 级 | D 级 |
|------|------|------|------|------|
| 确定性 | 送葬者直接验证 | 多重交叉印证 | 单一来源 | 未验证 |
| 时效性 | 当前信息 | 1-2 天前 | 3+ 天前 | 陈旧/过时 |
| 来源可靠性 | 已验证真角色 | 高度可信 | 声明合理 | 存疑 |

### 常见角色组合分析

**7 人局标准配置**：
- 5 好人（3 镇民 + 1 外来者 + 1 旅行者）
- 实际通常：3 镇民 + 1 外来者（无旅行者时）
- 2 邪恶（1 爪牙 + 1 恶魔）

**如果有男爵**：
- 3 好人（1 镇民 + 3 外来者）
- 3 邪恶（2 爪牙含男爵 + 1 恶魔）
- 总人数仍为 7，但分布改变

### 关键决策检查清单

**处决前检查**：
- [ ] 这次处决的信息价值是什么？
- [ ] 是否有更高价值的目标？
- [ ] 处决后可能的信息如何影响后续？
- [ ] 是否有风险（如可能是圣徒）？
- [ ] 团队是否支持这个决策？

**信息分享前检查**：
- [ ] 分享这个信息的时机是否合适？
- [ ] 分享后邪恶阵营如何应对？
- [ ] 是否需要保留部分信息作为后续验证？
- [ ] 如何呈现信息最有说服力？

**残局决战前检查**：
- [ ] 所有已知信息是否一致？
- [ ] 恶魔候选是否已经锁定？
- [ ] 盟友投票是否可靠？
- [ ] 是否有备选方案（如市长能力）？
- [ ] 是否考虑了所有可能的信息干扰？
